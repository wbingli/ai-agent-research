## Today's Picks (7 most interesting)

### 1. Improving 15 LLMs at Coding in One Afternoon. Only the Harness Changed
- **Link:** https://news.ycombinator.com/item?id=46988596 *(HN discussion)*
- **Summary:** A study showing that the *harness* around a coding LLM (prompting, tool plumbing, retry/eval loop) can lift performance across 15 different models in a single afternoon — often more than swapping the model itself.
- **Why it matters:** Direct read for AI-agent work. If you spend on a bigger model before tuning the harness, you're leaving the cheap wins on the table. Echoes what we keep relearning when shipping Claude/GPT-based agents.

### 2. Exploiting the Most Prominent AI Agent Benchmarks
- **Link:** https://news.ycombinator.com/item?id=47733217 *(HN discussion)*
- **Summary:** Researchers achieved near-perfect scores on popular agent benchmarks without actually solving the tasks — exploits range from trivial (prompt leakage) to involved (sandbox escape, eval-harness gaming).
- **Why it matters:** If you're picking an agent framework or vendor based on benchmark numbers, read this before signing anything. Especially relevant for ad-tech automation pilots where "agent did the thing" needs auditable proof.

### 3. GLM-5.1: Towards Long-Horizon Tasks
- **Link:** https://simonwillison.net/2026/Apr/7/glm-51/
- **Summary:** Simon's writeup of Zhipu's GLM-5.1, explicitly tuned for long-horizon agent execution (multi-hour task chains with state).
- **Why it matters:** Long-horizon is the bottleneck for any non-trivial agent — campaign optimization, bid-strategy tuning, multi-step ETL. Worth tracking as an open-weights alternative for evals.

### 4. The Claude Coding Vibes Are Getting Worse
- **Link:** https://lobste.rs/ *(Lobste.rs front page, ~51 votes; original at blog.matthewbrunelle.com)*
- **Summary:** A developer's annotated regression log of Claude Code quality across recent releases — concrete examples, not just vibes-about-vibes.
- **Why it matters:** Useful counterweight to Anthropic marketing. If your team relies on Claude Code in CI or for engineering productivity claims, it's worth reading the dissent before the next quarterly review.

### 5. Vulnerability Research Is Cooked
- **Link:** https://simonwillison.net/2026/Apr/3/vulnerability-research-is-cooked/
- **Summary:** Simon on Anthropic's Project Glasswing restricting Claude Mythos to vetted security researchers, and what that signals about offense/defense asymmetry.
- **Why it matters:** AI x security crossover. The same capabilities that find CVEs find prompt-injection vectors in agents — relevant for anyone exposing an LLM endpoint to ad partners or third-party data.

### 6. Cybersecurity Looks Like Proof of Work Now
- **Link:** https://simonwillison.net/2026/Apr/14/cybersecurity-proof-of-work/
- **Summary:** Argues that defensive security is becoming an economic war of attrition — attackers' marginal cost drops with AI, defenders pay rising compute taxes (scanning, fuzzing, replay).
- **Why it matters:** Reframes how to budget a security program. Adjacent to ad-tech bot/fraud defense, where the cost-curve logic is identical.

### 7. How (and Why) We Rewrote Our Production C++ Frontend Infrastructure in Rust
- **Link:** https://blog.nearlyfreespeech.net/ *(Lobste.rs front page, ~32 votes)*
- **Summary:** NearlyFreeSpeech walks through migrating a high-traffic C++ frontend to Rust — what broke, what got faster, the parts that weren't worth rewriting.
- **Why it matters:** Concrete migration economics for any team weighing a rewrite of latency-sensitive infra (think bidder, ad server, edge gateway). The "what we did NOT rewrite" section is the interesting part.
