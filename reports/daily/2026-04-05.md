# AI Agent Research Daily Digest — 2026-04-05

## 🔥 Top Discoveries

### 1. OpenSpace — Self-Evolving Skills for AI Agents
- **Repo:** https://github.com/HKUDS/OpenSpace
- **Released:** v0.1.0 (Apr 3, 2026)
- **What:** A skill evolution engine that plugs into any AI agent (Claude Code, Codex, Cursor, OpenClaw, etc.). Skills auto-fix when broken, auto-improve from successful patterns, and auto-learn from usage. Community-shared skill marketplace.
- **Key claims:** 46% fewer tokens, self-evolving skills, cross-agent experience sharing
- **Why interesting:** Addresses the fundamental "agents never learn" problem. Instead of reasoning from scratch every time, agents reuse and evolve successful patterns. The community sharing angle creates network effects — one agent's improvement benefits all. Quality monitoring tracks skill performance across tasks.
- **Novel idea:** Skills as living, evolving artifacts rather than static prompts.

### 2. Routa — Kanban-Based Multi-Agent Coordination
- **Repo:** https://github.com/phodal/routa
- **What:** A workspace-first multi-agent platform where a Kanban board IS the coordination layer. Decomposes tasks into cards, specialized agents pick up work as it flows: Backlog → Todo → Dev → Review → Done. Desktop app (Tauri) + web.
- **Stack:** TypeScript/Next.js + Rust/Axum, supports MCP/ACP/A2A protocols
- **Why interesting:** Refreshing take on multi-agent coordination. Instead of complex orchestration code, uses a visual Kanban metaphor that humans already understand. Specialized agents (Refiner, Orchestrator, Crafter, Guard, Reporter) mirror real team roles. The board is both project manager and communication bus.
- **Novel idea:** Project management paradigm (Kanban) as agent orchestration primitive.

### 3. Vercel agent-browser — Rust-Native Browser Automation for Agents
- **Repo:** https://github.com/vercel-labs/agent-browser
- **What:** Fast native Rust CLI for browser automation. Snapshot-based interaction model (accessibility tree with refs), click/fill/type by ref. Installs via npm, brew, or cargo.
- **Why interesting:** Vercel entering the agent browser space with a Rust-native approach. The ref-based interaction pattern (snapshot → click @e2) is clean and efficient. Auto-detects existing Chrome/Brave/Playwright installations. No Node.js required for the daemon — pure Rust performance.
- **Novel idea:** Combining Rust performance with accessibility-tree-based agent interaction for browser control.

### 4. bb-browser — Your Real Browser as the API
- **Repo:** https://github.com/epiral/bb-browser
- **What:** CLI + MCP server that lets AI agents control Chrome using your actual login state. 103 commands across 36 platforms (Twitter, GitHub, Zhihu, arXiv, YouTube, etc.). Runs eval inside your browser tab, uses your cookies.
- **Why interesting:** Completely inverts the browser automation paradigm. Instead of headless browsers that get blocked by anti-bot, it uses your REAL browser session. The website thinks it's you — because it IS you. Invisible to anti-bot detection. Has OpenClaw integration built in.
- **Novel idea:** Authenticated browsing-as-a-service using the user's own browser state.

### 5. pi-mono — AI Agent Toolkit (Trending: 31.7K ⭐)
- **Repo:** https://github.com/badlogic/pi-mono
- **Stars:** 31,713 (+340 today)
- **What:** Full-stack AI agent toolkit: coding agent CLI, unified LLM API, TUI & web UI libraries, Slack bot, vLLM pods. TypeScript monorepo.
- **Why interesting:** Massive traction (31K+ stars). Comprehensive monorepo approach — one toolkit covering CLI coding agent, LLM abstraction, UI components, and deployment. The "everything in one place" philosophy reduces integration friction.
