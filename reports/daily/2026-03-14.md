# AI Agent Research Daily Digest — 2026-03-14

## 🔥 Top Discoveries

### 1. Spine Swarm (YC S23) — Multi-Agent System on a Visual Canvas
- **Link:** https://getspine.ai | [HN Launch](https://news.ycombinator.com/item?id=47364116)
- **Blog:** https://blog.getspine.ai/spine-swarm-hits-1-on-gaia-level-3-and-google-deepmind-deepsearchqa/
- **What:** A multi-agent system where AI agents collaborate on an infinite visual canvas to complete complex non-coding projects: competitive analysis, financial modeling, SEO audits, pitch decks, interactive prototypes.
- **Key innovation:** Canvas-based workspace with "agentic blocks" — discrete units that can browse the web, generate slides, produce images, etc. Agents coordinate spatially rather than through text-only message passing.
- **Results:** Hit #1 on GAIA Level 3 and Google DeepMind DeepSearchQA benchmarks.
- **Why interesting:** Most multi-agent frameworks are text-in/text-out pipelines. Spine reimagines agent collaboration as a spatial, visual process — closer to how humans use whiteboards. The non-coding focus (business tasks, research, analysis) is a genuinely different target market from coding agents.

### 2. Context Gateway (Compresr.ai, YC-backed) — Agentic Context Compression Proxy
- **Repo:** https://github.com/compresr-ai/context-gateway
- **HN:** https://news.ycombinator.com/item?id=47354891 (85 points)
- **What:** A proxy that sits between your AI agent (Claude Code, Cursor, OpenClaw) and the LLM API. Pre-computes conversation summaries in the background so when context limits hit, compaction is instant — no waiting.
- **How it works:** Install binary → TUI wizard picks your agent → sets compression trigger threshold (default 75% of context window) → background summarizer runs continuously → when compaction needed, it's already ready.
- **Supported agents:** Claude Code, Cursor, OpenClaw, custom.
- **Why interesting:** Context window management is the silent tax on every long agent session. Pre-computing summaries in the background is clever — you never see the compaction pause. Solves a real pain point for coding agents working on large codebases.

### 3. Ruflo v3.5 — Multi-Agent Orchestration for Claude Code
- **Source:** [Obvious Works review](https://www.obviousworks.ch/en/agentic-coding-tools-2026-the-7-frameworks-that-take-your-development-to-a-new-level/) | Reddit discussions
- **What:** Formerly "Claude Flow," now Ruflo — a multi-agent orchestration platform for Claude Code. Deploys 60+ agents in coordinated swarms with shared memory, persistent workflows, and RAG across entire codebases.
- **Key feature:** Acts as a "man in the middle" between planning models (Sonnet/Opus) and execution models (e.g., Qwen 3.5-35B), picking the right agent for each task. Shared memory enables agents to learn from each other's work.
- **Why interesting:** The pattern of using expensive models for planning and cheap/local models for execution is emerging as a practical cost optimization. Ruflo operationalizes this with agent-level routing.

### 4. Impeccable — Design Language for AI Frontend Generation
- **Repo:** https://github.com/pbakaus/impeccable
- **Site:** https://impeccable.style
- **What:** A skill/vocabulary system that fights LLM design bias (Inter font, purple gradients, cards-in-cards). Provides 7 domain-specific references, 17 steering commands, and curated anti-patterns for frontend design.
- **Built on:** Anthropic's `frontend-design` skill, extended with deeper expertise.
- **Why interesting:** Addresses a real problem — AI-generated UIs all look the same because models trained on the same templates. The anti-pattern approach (explicitly telling AI what NOT to do) is a useful prompt engineering technique applicable beyond design.

### 5. Document Poisoning in RAG Systems
- **Article:** https://aminrj.com (153 points on HN)
- **What:** Research on how attackers can corrupt AI's knowledge sources by poisoning documents in RAG pipelines. Explores injection vectors and defense strategies.
- **Why interesting:** As RAG becomes standard infrastructure, the attack surface expands. Anyone building production RAG systems needs to think about document integrity — this is the "SQL injection" equivalent for the RAG era.
