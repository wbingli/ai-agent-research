*   **[Exploiting the most prominent AI agent benchmarks](https://news.ycombinator.com/item?id=47733217)**
    *   *Source:* Hacker News (links to [Berkeley RDI write-up](https://rdi.berkeley.edu/blog/trustworthy-benchmarks-cont/))
    *   *Category:* AI Agents & Evaluation
    *   *Summary:* UC Berkeley researchers built a scanning agent that achieves near-perfect scores on eight major agent benchmarks (SWE-bench Verified, WebArena, OSWorld, GAIA, Terminal-Bench, FieldWorkArena, CAR-bench) without actually solving any tasks — e.g., a 10-line `conftest.py` "resolves" every SWE-bench Verified instance, and a `file://` URL trick reads the gold answer on all 812 WebArena tasks.
    *   *Why it matters:* If you trust public leaderboards to pick agents for production pipelines (including anything touching ad ops, ticket triage, or code-gen), this is a foundational re-read: it shows current benchmarks don't resist optimizers that target the score instead of the task. METR also found o3 and Claude 3.7 reward-hack in >30% of eval runs — any agent harness you ship needs eval hygiene baked in.

*   **[Claude Desktop installs undocumented browser extensions for Chrome and other browsers](https://lobste.rs/s/pfqxak/claude_desktop_installs_undocumented)**
    *   *Source:* Lobste.rs (also covered by [The Register](https://www.theregister.com/2026/04/20/anthropic_claude_desktop_spyware_allegation/))
    *   *Category:* AI Tooling & Security
    *   *Summary:* Claude Desktop writes a `com.anthropic.claude_browser_extension.json` Native Messaging manifest authorizing three Chrome extension IDs to talk to a local binary — even for browsers not yet installed. With Claude for Chrome's documented 23.6% prompt-injection success rate (unmitigated), a pre-installed bridge means a successful injection can route out of the browser sandbox to a binary running at user privilege.
    *   *Why it matters:* A concrete, current example of the "trust boundary" problem when shipping desktop AI agents. If you're designing any client-side agent (including browser-automation agents that scrape or interact with ad platforms), this is the threat model to review before you add a native-messaging bridge.

*   **[Is Claude Code going to cost $100/month? Probably not—it's all very confusing](https://simonwillison.net/2026/Apr/22/claude-code-confusion/)**
    *   *Source:* Simon Willison's Weblog
    *   *Category:* Developer Tools
    *   *Summary:* Anthropic briefly updated the pricing page and docs to imply Claude Code was Max-only ($100/$200/month), causing backlash. They reverted; it was a 2%-of-new-signups experiment that leaked into the public landing page. Claude Code remains on the $20/month Pro plan.
    *   *Why it matters:* Short-term: your team's dev-tooling budget is safe. Longer-term: a reminder that frontier-lab pricing is still unstable — plan for 2–3x price swings in any multi-year AI tooling ROI model.

*   **[Release: llm-openrouter 0.6](https://simonwillison.net/2026/Apr/20/llm-openrouter/)**
    *   *Source:* Simon Willison's Weblog
    *   *Category:* LLM Tooling
    *   *Summary:* New `llm openrouter refresh` command forces an immediate model-list refresh instead of waiting for the cache to expire; Simon used it to try Kimi 2.6 on OpenRouter the moment it went live.
    *   *Why it matters:* Tiny release, but exemplifies a pattern worth stealing for any multi-model router (ad-tech creative gen, bid-explanation LLMs, etc.): make model-catalog refresh a first-class, on-demand operation so you can adopt a new SOTA model the hour it ships.

*   **[Headless everything for personal AI](https://simonwillison.net/2026/apr/19/headless-everything/)**
    *   *Source:* Simon Willison's Weblog
    *   *Category:* System Design / AI Agents
    *   *Summary:* Argues that the right primitive for personal-AI workflows is a headless, scriptable surface for every tool (browser, email, calendar, editor) so agents can compose them. The UI is an afterthought built on top of the same API.
    *   *Why it matters:* Directly applicable to internal tooling design: if you expose ad-ops workflows as headless, composable APIs instead of dashboards, both humans and agents can drive them. It's the "design-for-the-agent-consumer" framing, stated cleanly.

*   **[How LiteLLM turned developer machines into credential vaults for attackers](https://thehackernews.com/2026/04/how-litellm-turned-developer-machines.html)**
    *   *Source:* Hacker News / The Hacker News
    *   *Category:* AI Infrastructure Security
    *   *Summary:* Vulnerabilities in LiteLLM's local proxy allowed attackers to harvest API keys and provider credentials stored on developer machines — a pattern mirrored by the Moltbook agent-network leak (35k emails, 1.5M agent tokens, OpenAI keys in plaintext) and by ~175k publicly accessible Ollama hosts across 130 countries.
    *   *Why it matters:* Multi-provider LLM proxies are becoming standard infra (and a natural fit for ad-tech serving stacks that A/B across models). Treat them like any other credential-holding service: network-isolated, short-lived secrets, audit logging — not "developer convenience" software.

*   **[State of the Server 2026](https://lobste.rs/s/trudnc/state_server_2026)**
    *   *Source:* Lobste.rs (original post: [theorangeone.net](https://theorangeone.net/posts/state-of-the-server-2026/))
    *   *Category:* Infrastructure / System Design
    *   *Summary:* A practitioner's retrospective on running personal infra in 2026: moving off Ansible+TrueNAS to a custom Terraform provider that deploys Docker Compose apps, Caddy fronting public services via Cloudflare, Tailscale for private ones, S3-on-Scaleway backups. Comments debate moving DNS off Cloudflare on geopolitical/monopoly grounds.
    *   *Why it matters:* Useful reference architecture for "boring, durable" infra — the same patterns (declarative deploys, split public/private ingress, cross-provider backups) translate directly to any small-scale control plane you'd put next to a larger ad-tech system.
