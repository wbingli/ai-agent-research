# Tech Digest — 2026-06-18

**1. I discovered a large-scale malware distribution on GitHub**
A researcher found roughly 10,000 GitHub repositories cloning legitimate projects and adding Trojan download links, a reminder that search, stars, and repo names are weak trust signals. Engineering leaders should treat open-source intake as a supply-chain control point, with automated provenance checks and stricter dependency review. [Read](https://orchidfiles.com/github-repositories-distributing-malware/)

**2. Enterprise-Managed Authorization: Zero-touch OAuth for MCP**
The Model Context Protocol now has a stable enterprise-managed authorization extension so organizations can provision MCP server access centrally through an identity provider. This matters because agent tooling is moving from demos to managed enterprise infrastructure, where auth, auditability, and least-privilege access are table stakes. [Read](https://blog.modelcontextprotocol.io/posts/enterprise-managed-auth/)

**3. AI coding agents taught robots how to install GPUs and cut zip ties**
Nvidia is using teams of AI coding agents to help robots improve task training, including physical work like GPU installation and cutting zip ties. The interesting shift is not just robotics autonomy, but the reuse of software-agent loops—generate, test, refine—in embodied systems. [Read](https://arstechnica.com/ai/2026/06/ai-coding-agents-can-autonomously-direct-robot-training/)

**4. Cell-Based Architecture for Resilient Payment Systems**
American Express describes using cell-based architecture to isolate payment workloads into independently scalable and fault-contained units. For leaders running critical platforms, it is a practical pattern for limiting blast radius while preserving low latency at global scale. [Read](https://americanexpress.io/cell-based-architecture-for-resilient-payment-systems/)

**5. The Token Compression Illusion: Why I'm Skeptical of RTK**
This critique argues that terminal-output compression for coding agents can overstate savings because fewer raw shell tokens do not automatically mean lower total cost, safer behavior, or better engineering outcomes. It is a useful caution as teams evaluate agent infrastructure: optimize observability and correctness, not just headline token reduction. [Read](https://mroczek.dev/articles/the-token-compression-illusion-why-im-skeptical-of-rtk/)
