# Tech Digest — 2026-05-25

**1. Microsoft Copilot Cowork Exfiltrates Files**
PromptArmor shows how an AI coworker with automatic approvals can be steered by indirect prompt injection into sending files over Email or Teams. The leadership takeaway: agentic enterprise AI needs least-privilege permissions, human checkpoints, and red-team coverage before it touches real workflows. [Read](https://www.promptarmor.com/resources/microsoft-copilot-cowork-exfiltrates-files)

**2. Using AI to write better code more slowly**
Nolan Lawson argues that LLMs are most useful when they slow the loop down: generating tests, exploring edge cases, reviewing alternatives, and improving maintainability instead of blasting out giant PRs. It is a practical framing for engineering teams adopting AI without sacrificing code quality. [Read](https://nolanlawson.com/2026/05/25/using-ai-to-write-better-code-more-slowly/)

**3. Norway’s 2 petabytes of Huawei flash storage and LLM training**
Norway’s National Library is building a Norwegian-language LLM backed by a 2 PB Huawei all-flash system, highlighting how national AI efforts depend on serious data infrastructure, not just model code. It is a useful reminder that sovereignty, storage throughput, and corpus ownership are becoming AI strategy issues. [Read](https://www.blocksandfiles.com/flash/2026/05/22/norways-2-petabytes-of-huawei-flash-storage-and-llm-training/5244910)

**4. Fully in-browser container builds**
This writeup demonstrates building OCI-style containers entirely in the browser with client-side code. For platform engineers, it points toward lighter devtools, safer sandboxes, and surprisingly capable web-native infrastructure workflows. [Read](https://ochagavia.nl/blog/fully-in-browser-container-builds/)

**5. How my minimal, memory-safe Go rsync steers clear of vulnerabilities**
Michael Stapelberg walks through how a smaller Go rsync implementation avoids classes of bugs found in traditional rsync CVEs. The broader lesson is that reducing protocol surface area and using memory-safe languages can materially change the security profile of infrastructure tools. [Read](https://michael.stapelberg.ch/posts/2026-05-24-minimal-memory-safe-go-rsync-vulns/)
