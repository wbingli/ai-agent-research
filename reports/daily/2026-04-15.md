# Daily AI Research Digest - 2026-04-15

## Summary
Today’s strongest pattern is AI tooling moving from generic assistants toward more opinionated infrastructure around coding agents: persistent memory, repo-native structural context, specialized multi-agent orchestration, and MCP-native observability/security layers. The most interesting finds were not the biggest repos, but the ones adding new primitives around agent workflow rather than just wrapping an LLM.

## Top Finds

### 1. Buddy
- **Link:** https://github.com/fiorastudio/buddy
- **Stars:** 23
- **Created:** 2026-04-09
- **Main idea:** A persistent cross-client AI companion that revives the old Claude `/buddy` concept as an MCP server.
- **How it works:** Stores companion state in local SQLite, tracks XP/mood/species/personality stats, and plugs into Claude Code, Codex CLI, Cursor CLI, Copilot CLI, Gemini CLI, and other MCP-capable clients.
- **Why interesting:** Most agent tooling is utility-first. Buddy is unusual because it treats long-term emotional continuity and lightweight feedback loops as product primitives, then makes them portable across terminals via MCP. It is a good example of MCP being used for agent UX, not just external tools.

### 2. Jinju Legal Orchestrator
- **Link:** https://github.com/kipeum86/legal-agent-orchestrator
- **Stars:** 27
- **Created:** 2026-04-11
- **Main idea:** A domain-specific multi-agent law firm built on Claude Code, with eight specialist legal agents and explicit audit trails.
- **How it works:** A lead orchestrator classifies legal requests, chooses collaboration mode (sequential handoff, parallel research, debate), dispatches to specialist repos unchanged, logs execution to `events.jsonl`, and produces a consolidated `case-report.md`.
- **Why interesting:** This is a concrete verticalization of coding-agent style orchestration into legal workflows. The notable part is not “multiple agents” itself, but the strong auditability model, specialist isolation, and domain-grade deliverables. It feels closer to a real workflow engine than a demo swarm.

### 3. Supermodel CLI
- **Link:** https://github.com/supermodeltools/cli
- **Stars:** 23
- **Created:** 2026-03-31
- **Main idea:** Generate local code graph shard files so any coding agent can infer call graphs, dependencies, and blast radius without new integrations.
- **How it works:** Uploads the repo for analysis, then writes plain-text `.graph` or split `.calls/.deps/.impact` files next to source files. Agents consume them through ordinary file reads, grep, and cat.
- **Why interesting:** This is a clever “agent compatibility” trick. Instead of requiring every agent vendor to support a new protocol, it pushes structural context into the file system where existing agents already know how to look. That makes it a practical bridge between code intelligence and today’s file-centric coding agents.

### 4. claude-cortex
- **Link:** https://github.com/rudymartinezai/claude-cortex
- **Stars:** 1
- **Created:** 2026-04-15
- **Main idea:** Local-first memory system for Claude Code with automatic capture, semantic recall, and a temporal knowledge graph.
- **How it works:** Hooks into Claude Code session lifecycle events, stores captures locally, indexes them into ChromaDB, and exposes recall/knowledge-graph tools back through MCP.
- **Why interesting:** The implementation is more ambitious than a simple vector memory plugin. The layered memory stack, PreCompact reinjection, and explicit KG model point toward a stronger answer to session continuity than the usual “stuff transcripts in a vector DB” approach.

### 5. trace-mcp
- **Link:** https://github.com/nikolai-vysotskyi/trace-mcp
- **Stars:** 18
- **Created:** 2026-04-03
- **Main idea:** Framework-aware code intelligence MCP server that gives agents request-flow tracing, dependency impact, decision memory, and cross-session context.
- **How it works:** Builds a cross-language graph across frameworks, ORMs, UI libraries, and session history, then exposes high-level tools like `get_change_impact`, `get_request_flow`, and `plan_turn` over MCP.
- **Why interesting:** The key innovation is framework semantics, not just static code parsing. It tries to answer “what actually happens in this app?” across route, controller, ORM, UI, and past decisions, which is exactly where many coding agents still struggle.

## Additional Noteworthy Finds

### skillgrab
- **Link:** https://github.com/briascoi/skillgrab
- **Stars:** 2
- **Main idea:** Auto-detects a project stack and installs matching skills from skills.sh.
- **How it works:** Scans package manifests, Dockerfiles, and README content, queries the skills registry, ranks/dedupes/validates candidate skills, then installs them in grouped batches.
- **Why interesting:** Feels like package-manager ergonomics for agent skills. If skill ecosystems keep growing, auto-selection becomes an important missing layer.

### claude-token-lens
- **Link:** https://github.com/wassimbensalem/claude-token-lens
- **Stars:** 4
- **Main idea:** Live token attribution for Claude Code usage by tool, agent, skill, and MCP server.
- **How it works:** Reads Claude Code session files directly and renders a live dashboard showing rolling output-token burn, source attribution, and quota estimates.
- **Why interesting:** Good observability tooling for agent cost control. This is increasingly important as workflows become multi-agent and MCP-heavy.

### trawl
- **Link:** https://github.com/jeffdhooton/trawl
- **Stars:** 1
- **Main idea:** Local-first intelligent web scraping CLI for AI agents.
- **How it works:** Routes each URL through the cheapest valid extraction path, from plain HTTP to headless Chromium to PDF extraction/OCR, remembers per-host routing, and outputs clean markdown/JSONL.
- **Why interesting:** A pragmatic local alternative to hosted scrape APIs, with good fit for agent pipelines that need deterministic, resumable, Unix-friendly content extraction.

## Takeaways
- **Memory is becoming infrastructure.** claude-cortex and trace-mcp both push beyond naive transcript recall toward structured persistence.
- **Repo structure is the next battleground.** Supermodel and trace-mcp both try to compress architectural understanding into agent-readable form.
- **MCP is broadening fast.** Today’s notable MCP uses span companions, code intelligence, legal workflows, observability, and security.
- **Vertical agents are maturing.** Jinju Legal Orchestrator shows a serious domain-specific workflow with auditability and specialist isolation.
