# AI Agent Research Daily Digest — March 24, 2026

## 🔥 Top Discoveries

### 1. GitAgent Launches on Product Hunt — "Docker for AI Agents" Goes Live
- **Link**: [producthunt.com/products/gitagent-2](https://www.producthunt.com/products/gitagent-2) | [GitHub](https://github.com/lyzr-ai/gitagent)
- **By**: Lyzr AI
- **What**: Framework-agnostic, git-native standard for defining AI agents. Extracts agent config, logic, tools, memory into a portable, version-controlled directory structure (`agent.yaml`, `SOUL.md`, `DUTIES.md`, `skills/`, `tools/`, `rules/`). Export to LangChain, AutoGen, CrewAI, or Claude Code with one command.
- **Why interesting**: Launched on Product Hunt today. The file structure (SOUL.md, DUTIES.md, rules/) mirrors what OpenClaw/Clawd already uses — this could become the interchange format. Also notable: `gitagent validate` checks agent configs against safety rules before deployment, enforcing separation of duties. If this gains traction, it standardizes the "agent definition" layer.

### 2. Autoresearch in Practice — Using Karpathy's Loop on Real Research
- **Link**: [ykumar.me/blog/eclip-autoresearch](https://ykumar.me/blog/eclip-autoresearch/) | 391 points on HN
- **What**: A researcher applied Karpathy's Autoresearch pattern (constrained optimization loop with LLM agent) to their old eCLIP research. The agent iteratively edits `train.py`, runs sandboxed experiments (~5 min each), evaluates, commits or reverts, and repeats. Used Claude Code as the agent, with tight sandboxing (no network, no pip, containerized training).
- **Why interesting**: This is the best real-world Autoresearch writeup I've seen — practical, honest about what worked and didn't. The sandboxing approach (containerized training, locked-down Claude Code permissions) is a pattern worth adopting. Shows AI-assisted research is moving from demos to real workflows.

### 3. RAXE Research Radar — Agent Skill Supply Chain Is Broken
- **Link**: [raxe.ai/labs/radar/radar-2026-002](https://raxe.ai/labs/radar/radar-2026-002)
- **Papers**: RAP-2026-005 (ArXiv 2603.16572), RAP-2026-008, RAP-2026-009
- **What**: Largest empirical study of the AI agent skill ecosystem (238,180 skills from ClawHub, Skills.sh, etc.). Key findings:
  - Five scanners disagree wildly on malicious skill rates (3.79% to 41.93%)
  - Only 0.12% flagged by ALL five scanners
  - Real attack vector discovered: adversaries hijack abandoned GitHub repos indexed by skill marketplaces, silently replacing legitimate skills
  - Only tool whitelisting and privilege separation materially reduced attack success
- **Why interesting**: Directly relevant to anyone using agent skill marketplaces. The supply-chain hijack via abandoned GitHub usernames is a real, exploitable vulnerability (121 skills across 7 repos). This is the agent ecosystem's "left-pad" moment waiting to happen.

### 4. GitHub Copilot Squad — Multi-Agent Dev Teams in Your Repo
- **Link**: [github.blog/ai-and-ml/github-copilot/how-squad-runs-coordinated-ai-agents-inside-your-repository](https://github.blog/ai-and-ml/github-copilot/how-squad-runs-coordinated-ai-agents-inside-your-repository/) | [github.com/bradygaster/squad](https://github.com/bradygaster/squad)
- **What**: Open source project built on GitHub Copilot. Two commands (`npm install -g @bradygaster/squad-cli` + `squad init`) drops a specialized AI team (lead, frontend dev, backend dev, tester) directly into your repo. Repository-native orchestration — no external vector databases or framework setup.
- **Why interesting**: The zero-setup multi-agent approach is compelling. Instead of building orchestration layers, you get pre-configured role-based agents that work within your existing repo. This is GitHub's answer to the "agent harness" trend (superpowers, gstack, agency-agents).

### 5. a16z: AI Agents Will Kill Internet Advertising
- **Link**: [forklog.com](https://forklog.com/en/a16z-predicts-the-end-of-internet-advertising-due-to-ai-agents/)
- **What**: a16z predicts that AI agents performing tasks on behalf of users will fundamentally break the internet advertising model. When agents browse, shop, and transact — they don't see ads. Open protocols and status codes will replace the advertising hack.
- **Why interesting**: Directly relevant to Wenbing's work at Disney Streaming AD Platform. If agents become the primary browsing interface, ad-supported streaming models need to evolve. Worth tracking this thesis.
