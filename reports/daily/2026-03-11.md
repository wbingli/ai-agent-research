# AI Agent Research Daily Digest — March 11, 2026

## 🔥 Top Discoveries

### 1. Karpathy's `autoresearch` — Autonomous ML Research Agent
- **Link:** https://github.com/karpathy/autoresearch
- **Stars:** Rapidly growing (thousands in first week)
- **What it is:** A deliberately minimal (~630 lines) framework that lets an AI agent autonomously run ML experiments on a single GPU overnight. The agent modifies `train.py`, runs 5-minute training cycles, evaluates BPB (bits per byte), and only commits improvements via git.
- **How it works:** Three files — `prepare.py` (fixed data prep), `train.py` (the agent edits this), and `program.md` (instructions for the AI agent, i.e. "programming the program"). The agent loop: edit code → train → evaluate → keep/discard → repeat. Everything is fair game: architecture, hyperparameters, optimizer, batch size.
- **Why interesting:** Karpathy frames this as the origin story of autonomous AI research. The key insight is "you're not touching Python files like a researcher — you're programming the `program.md` that guides the AI." It's a meta-research framework. The deliberately small scope makes it reproducible on consumer hardware. Based on his nanochat training setup.
- **Karpathy's narrative:** Includes a fictional prologue about AI agents eventually running all research, code becoming "self-modifying binary beyond human comprehension" — classic Karpathy storytelling.

### 2. GitHub Copilot SDK — Embeddable Agentic Execution Engine
- **Link:** https://github.com/github/copilot-sdk/
- **Blog:** https://github.blog/ai-and-ml/github-copilot/the-era-of-ai-as-text-is-over-execution-is-the-new-interface/
- **Released:** March 10, 2026
- **What it is:** GitHub released the Copilot SDK, allowing developers to embed the same planning and execution engine that powers Copilot CLI directly into their own applications. Not just text generation — full agentic execution (plan steps, invoke tools, modify files, recover from errors).
- **Why interesting:** This is a major shift. GitHub is essentially saying "AI as text output is over" and providing an embeddable agentic runtime. Any app that can trigger logic can now trigger agentic execution. This competes directly with LangChain/CrewAI by offering a production-tested orchestration engine.

### 3. "The Agency" — 61-Agent Company Simulation (10K Stars in 7 Days)
- **Link:** Referenced in Medium article (GitHub repo with 61 agent definition files)
- **Source:** https://medium.com/coding-nexus/someone-built-a-full-ai-agency-on-github-61-agents-10k-stars-in-7-days-ac976f85925d
- **What it is:** A GitHub repo containing 61 agent definition files across 9 "divisions" — engineers, designers, growth marketers, product managers. Each agent has a specific identity, workflow, deliverables, and success metrics. Works with Claude Code via simple file copy.
- **How it works:** Copy agent files to `~/.claude/agents/`, then activate any agent in Claude Code sessions. Not generic prompts — actual agent personalities with processes built in. Conversion scripts available for other tools.
- **Why interesting:** This is prompt engineering meets organizational design. Instead of one-size-fits-all AI, it structures AI like a real company with specialized roles. 10K stars in a week suggests massive demand for structured agent personas.

### 4. Copilot Code Review Goes Agentic (GA)
- **Link:** https://github.blog/changelog/2026-03-05-copilot-code-review-now-runs-on-an-agentic-architecture/
- **Released:** March 5, 2026 (GA)
- **What it is:** GitHub Copilot's code review feature now runs on a full agentic tool-calling architecture. Available for Pro, Pro+, Business, and Enterprise tiers.
- **Why interesting:** Code review is one of the most natural fits for agentic AI — it needs to read code, understand context, check for patterns, and suggest fixes. Moving from static analysis to agentic tool-calling is a significant upgrade.
