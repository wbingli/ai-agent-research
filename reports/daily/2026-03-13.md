# AI Agent Research Daily Digest — 2026-03-13

## 🔥 Top Discoveries

### 1. Hermes Agent v0.2.0 (NousResearch)
- **Repo:** https://github.com/NousResearch/hermes-agent
- **What:** Open-source agentic coding platform from Nous Research, rapidly becoming a serious alternative to closed-source agent harnesses.
- **Key features in v0.2:** Full MCP client support, ACP server for editor integration, filesystem checkpoints with rollback, git worktree isolation, subagent transparency, local browser support. Supports multiple providers (Claude, GLM, Kimi, MiniMax, OpenAI).
- **Why interesting:** It's model-agnostic, open-source, and shipping at a pace that's attracting real migration from other platforms. The Hermes IDE — an AI-native terminal with isolated git worktrees and a prompt composer — extends it further. This is the most complete open-source agent harness right now.
- **Community:** Active r/LocalLLaMA discussion, growing adoption.

### 2. GitHub Copilot SDK — Programmable Agentic Execution
- **Link:** https://github.com/github/copilot-sdk
- **Blog:** https://github.blog/ai-and-ml/github-copilot/the-era-of-ai-as-text-is-over-execution-is-the-new-interface/
- **What:** GitHub released an SDK that lets you embed Copilot's agentic execution engine directly into your own applications. Not just chat completions — full multi-step planning, tool invocation, file modification, and error recovery.
- **Key patterns:** (1) Delegate multi-step work to agents, (2) Ground execution in structured runtime context via MCP, (3) Observable constrained execution.
- **Why interesting:** This is GitHub saying "the era of AI as text is over." The SDK turns Copilot from an IDE feature into embeddable infrastructure. Combined with MCP for tool integration, this could become the default way apps get agentic capabilities.

### 3. GitHub Copilot Agentic Capabilities GA in JetBrains
- **Link:** https://github.blog/changelog/2026-03-11-major-agentic-capabilities-improvements-in-github-copilot-for-jetbrains-ides/
- **What:** Custom agents, sub-agents, plan agent now GA in JetBrains IDEs. Agent hooks in preview. Auto-approve support for MCP tools.
- **Why interesting:** Relevant for Wenbing's Java/IntelliJ workflow at Disney. Sub-agents and plan agent could significantly improve complex refactoring workflows.

### 4. CursorBench — New Coding Agent Evaluation Methodology
- **Source:** Cursor team (via @cursor_ai)
- **What:** New benchmark combining offline benchmarks with online request-derived metrics to score coding models on both intelligence and token efficiency. GPT-5.4 leads on correctness with efficient token usage.
- **Why interesting:** Public coding benchmarks are saturating. CursorBench introduces multi-axis measurement (correctness, token efficiency, interaction behavior, real task fit) — a more honest evaluation of what matters in practice.

### 5. Google Gemini Embedding 2 — First Natively Multimodal Embedding Model
- **Source:** Google AI Studio
- **What:** Maps text, images, audio, video, and PDFs into a single vector space. Supports Matryoshka Representation Learning for flexible output dimensions.
- **Why interesting:** Enables true multimodal RAG — search across document types in a single embedding space. Game-changer for retrieval systems that deal with mixed media.

### 6. Mixedbread Wholembed v3 — SOTA Multimodal Retrieval
- **Source:** @mixedbreadai
- **What:** Claims SOTA retrieval across multiple modalities, competing directly with Gemini Embedding 2.
- **Why interesting:** Two strong multimodal embedding models dropping in the same week signals a new wave of multimodal RAG tooling.

## 📊 Trends & Analysis

### The "Harness" Is the Product Now
Multiple converging signals this week: Harrison Chase (LangChain) explicitly framing agent value around harnesses, sandboxes, memory, and observability rather than model quality alone. The Copilot SDK, Hermes Agent, and Codex Automations GA all point the same direction — **the orchestration layer around the model is where differentiation happens**.

### MCP Is Baseline Infrastructure
Despite "MCP is dead" jokes circulating, Uber is using MCP internally, GitHub's Copilot SDK uses MCP for tool integration, and Hermes Agent shipped full MCP client support. MCP isn't novel anymore — it's plumbing. The debate has shifted from "should we use MCP?" to "how do we make MCP work reliably in production?"

### Coding Agent Bifurcation
Interesting split emerging: (1) Fully autonomous agents (Codex Automations GA with worktree/branch choice, reusable templates) vs. (2) "Stay in the loop" tooling where fast inline autocomplete outperforms agentic workflows for understanding preservation. ThePrimeagen argues the second camp hard.

### OpenAI Codex Automations GA
- Shipped worktree vs. branch choice, model/reasoning controls, and reusable templates
- Moving from "coding agent" to "coding automation platform"
