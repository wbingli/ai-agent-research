# Tech Digest — 2026-05-14

**1. Rewrite Bun in Rust has been merged**
Bun’s long-running rewrite moves major internals from Zig toward Rust, a notable bet on memory safety, ecosystem leverage, and maintainability for a high-performance JavaScript runtime. For engineering leaders, it is a reminder that runtime/platform teams may accept large migrations when hiring, tooling, and long-term velocity outweigh short-term churn. [Read](https://github.com/oven-sh/bun/pull/30412)

**2. Codex is now in the ChatGPT mobile app**
OpenAI is putting Codex workflows into ChatGPT on mobile, pushing software agents closer to always-available, asynchronous engineering assistants rather than desktop-only tools. The strategic point is less “coding on a phone” and more that code review, task triage, and agent supervision are becoming ambient management loops. [Read](https://openai.com/index/work-with-codex-from-anywhere/)

**3. First public macOS kernel memory corruption exploit on Apple M5**
A security team describes building a working macOS kernel memory-corruption exploit against Apple’s newest hardware/software defenses in days. Even with strong platform mitigations, kernel-class bugs remain high-impact, so security programs still need rapid patching, exploit-aware threat modeling, and defense in depth. [Read](https://blog.calif.io/p/first-public-kernel-memory-corruption)

**4. PostgreSQL 18.4, 17.10, 16.14, 15.18, and 14.23 released**
PostgreSQL shipped security updates across all supported branches, closing 11 CVEs and fixing reliability bugs. Database patch windows matter: when the affected component is core infrastructure, delayed minor-version upgrades become operational risk, not routine backlog. [Read](https://www.postgresql.org/about/news/postgresql-184-1710-1614-1518-and-1423-released-3297/)

**5. What’s in a GGUF, besides the weights — and what’s still missing?**
This deep dive explains the metadata a local LLM runtime needs beyond raw weights, including tokenizer behavior, templates, architecture fields, and the gaps that still make model portability fragile. It is useful for anyone operationalizing local inference: “model file” standards are becoming deployment interfaces, not just packaging details. [Read](https://nobodywho.ooo/posts/whats-in-a-gguf/)
