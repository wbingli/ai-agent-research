# Tech Digest — 2026-06-16

**1. Running local models is good now**
Local LLMs have crossed from novelty into useful day-to-day engineering tools, especially for coding workflows on commodity Apple Silicon and small-server setups. For leaders, this changes the privacy, latency, cost, and vendor-risk tradeoffs around where AI-assisted development should run. [Read](https://vickiboykis.com/2026/06/15/running-local-models-is-good-now/)

**2. Agentic Code Review**
Addy Osmani argues that as coding agents get better, the highest-leverage engineering skill shifts from generating code to deciding what to trust, test, and merge. The practical takeaway is that teams need stronger review systems, not just faster code production. [Read](https://addyosmani.com/blog/agentic-code-review/)

**3. Iroh 1.0: Dial Keys, not IPs**
Iroh 1.0 proposes a networking abstraction where apps connect to stable cryptographic keys instead of brittle IP addresses, handling NAT traversal, mobility, and direct connections underneath. This is worth watching for distributed apps, edge devices, real-time sync, and media systems that need simpler peer connectivity. [Read](https://www.iroh.computer/blog/v1)

**4. A backdoor in a LinkedIn job offer**
A fake recruiter sent a “technical assessment” repo with a hidden remote-code-execution payload, showing how hiring workflows are becoming a supply-chain attack surface. Engineering orgs should treat take-home repos and AI-assisted dependency installs with the same sandboxing discipline as untrusted production artifacts. [Read](https://roman.pt/posts/linkedin-backdoor/)

**5. How memory safety CVEs differ between Rust and C/C++**
This post explains why memory-safety vulnerability reporting looks different in Rust than in C/C++, especially around unsafe blocks, library boundaries, and exploitability. It is a useful framing for security leaders deciding where Rust actually reduces risk and where process still matters. [Read](https://kobzol.github.io/rust/2026/06/15/how-memory-safety-cves-differ-between-rust-and-c-cpp.html)
