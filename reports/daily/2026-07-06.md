# Tech Digest — 2026-07-06

**1. GLM 5.2 and the coming AI margin collapse**
GLM 5.2 is framed as an open-weights model competitive with frontier coding/agent models at roughly 15–20% of the price. If this holds up, engineering leaders should expect faster price compression in inference, agent platforms, and vendor margins. [Read](https://martinalderson.com/posts/the-upcoming-ai-margin-collapse-part-1-glm-5-2/)

**2. A global workspace in language models**
Anthropic reports interpretability evidence that Claude uses an internal “global workspace” to coordinate information across tasks. This matters because better mechanistic understanding could improve debugging, safety reviews, and confidence in deploying LLMs into higher-autonomy workflows. [Read](https://www.anthropic.com/research/global-workspace)

**3. Better Models: Worse Tools**
Armin Ronacher argues that newer, smarter models can still regress badly in tool-calling behavior, making agents less reliable despite stronger raw intelligence. The operational lesson is to evaluate model upgrades against end-to-end workflows, not just benchmarks or chat quality. [Read](https://lucumr.pocoo.org/2026/7/4/better-models-worse-tools/)

**4. Januscape: Guest-to-Host Escape in KVM/x86**
Januscape is a disclosed KVM/x86 guest-to-host escape with proof-of-concept material, a reminder that VM isolation remains a critical security boundary. Teams running multi-tenant virtualization, CI sandboxes, or untrusted workloads should track patches and exposure quickly. [Read](https://github.com/V4bel/Januscape)

**5. PREEMPT_NONE Is Dead; Your Postgres Probably Doesn’t Care**
A Linux scheduler change prompted concern about Postgres throughput, but this analysis shows the scary benchmark result was driven by setup choices like missing huge pages. It is a useful reminder to validate performance claims against production-like configs before changing kernel or database strategy. [Read](https://thebuild.com/blog/preempt_none-is-dead-your-postgres-probably-doesnt-care/)
