# Tech Digest — 2026-05-17

**1. Show HN: Semble – Code search for agents that uses 98% fewer tokens than grep**
Semble is a code-search tool aimed at agent workflows, returning compact, relevant context instead of dumping whole files through grep/read loops. This matters because token budget and context quality are now core infrastructure constraints for production coding agents. [Read](https://github.com/MinishLab/semble)

**2. Recent Kernel exploits, attack surface reduction, example IPSEC**
This oss-security post walks through recent Linux kernel exploitation patterns and uses IPsec as a concrete example of reducing exposed attack surface. It is a useful reminder for engineering leaders that “enabled by default” kernel/network features can silently become fleet-wide risk. [Read](https://www.openwall.com/lists/oss-security/2026/05/16/3)

**3. DeepSeek-V4-Flash means LLM steering is interesting again**
Sean Goedecke argues that stronger local/open models make activation steering newly practical: instead of only prompting, engineers may soon tune behavior through model-internal “sliders.” If this matures, it could change how teams customize agent behavior, latency, and safety without full fine-tuning. [Read](https://www.seangoedecke.com/steering-vectors/)

**4. Native all the way, until you need text**
A macOS developer explains why rich, selectable, streaming Markdown chat UIs still push teams toward WebKit or Electron even when the rest of the app is native. For AI-product teams, text rendering is no longer a boring widget—it is a major UX and architecture decision. [Read](https://justsitandgrin.im/posts/native-all-the-way-until-you-need-text/)

**5. I don’t think AI will make your processes go faster**
Frederick Vanbrabant cautions that AI will not automatically accelerate broken organizational processes; it often just makes local steps faster while leaving queues, handoffs, and decision bottlenecks intact. The leadership takeaway: redesign the system before measuring AI ROI. [Read](https://frederickvanbrabant.com/blog/2026-05-15-i-dont-think-ai-will-make-your-processes-go-faster/)
