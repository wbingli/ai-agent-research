### 2. [AI Agents] Computer Use Agents 2026: Claude vs OpenAI vs Gemini
* **Source:** Hacker News discussion (DigitalApplied)
* **URL:** [https://www.digitalapplied.com/blog/computer-use-agents-2026-claude-openai-gemini-matrix](https://www.digitalapplied.com/blog/computer-use-agents-2026-claude-openai-gemini-matrix)
* **Why it matters:** Three different architectural bets are now in the field: OpenAI's Codex Background Computer Use (shipped Apr 16, macOS-first desktop automation), Anthropic's portable tool use across VMs/containers/remote desktops, and Google's browser-native Gemini Computer Use from Project Mariner. Useful mental model for where "agents do real work" is heading — and which substrate to target if you are building integrations or eval harnesses.

### 3. [AI/Security] Show HN: Runtime security for AI agents (injection, tool abuse, data exfiltration)
* **Source:** Hacker News
* **URL:** [https://news.ycombinator.com/item?id=47799856](https://news.ycombinator.com/item?id=47799856)
* **Why it matters:** Open-source runtime guardrails for agents that call tools and touch data — prompt-injection detection, tool-call policy enforcement, exfil tripwires. As agentic systems move into production (customer support, internal ops, ad-tech audience workflows), this is the control plane your platform team will eventually need. Worth scanning the threat model even if you do not adopt the project.
