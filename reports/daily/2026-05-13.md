# Tech Digest — 2026-05-13

**1. Achieving NGINX Remote Code Execution via an 18-Year-Old Vulnerability**
Researchers used automated source analysis to find multiple NGINX memory-corruption issues, including a critical heap overflow in rewrite/set handling. It is a reminder that mature infrastructure still hides exploitable edge cases, and that AI-assisted code auditing is becoming operationally useful. [Read](https://depthfirst.com/research/nginx-rift-achieving-nginx-rce-via-an-18-year-old-vulnerability)

**2. Show HN: Needle — Distilling Gemini Tool Calling into a 26M Model**
Needle is a tiny function-calling model distilled from Gemini that can run on very small devices. If the quality holds up, it points toward cheaper local agents, embedded automation, and lower-latency tool use without sending every decision to a frontier model. [Read](https://github.com/cactus-compute/needle)

**3. How and Why rqlite Takes Control of the SQLite Write-Ahead Log**
rqlite explains why a Raft-backed SQLite database needs to manage WAL checkpointing itself rather than leaving it entirely to SQLite. The writeup is a useful case study in the boundary between a local storage engine and distributed-system guarantees. [Read](https://philipotoole.com/how-and-why-rqlite-takes-control-of-the-sqlite-write-ahead-log/)

**4. Ardent — Database Branching for Coding Agents**
Ardent offers near-instant Postgres branches so coding agents can test migrations and data changes against isolated database copies. This is the kind of infrastructure primitive agentic development needs: fast, disposable, production-like state with much lower blast radius. [Read](https://www.tryardent.com/)

**5. Reverting the Incremental GC in Python 3.14 and 3.15**
Python core developers plan to roll back the new incremental garbage collector after production reports of significantly higher memory pressure. For engineering teams, it is a good example of why runtime upgrades need representative workload testing, not just benchmark wins. [Read](https://discuss.python.org/t/reverting-the-incremental-gc-in-python-3-14-and-3-15/107014)
