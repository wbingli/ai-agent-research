# Tech Digest — 2026-07-19

**1. Inkling: Thinking Machines’ first open-weights model**

Inkling is a 975B-parameter multimodal mixture-of-experts model with 41B active parameters, a 1M-token context window, controllable reasoning effort, and fine-tuning through Tinker. Its combination of open weights and practical customization makes it notable for teams evaluating adaptable foundation models rather than closed APIs. [Read](https://thinkingmachines.ai/news/introducing-inkling/)

**2. Claude Code is quietly shipping a Rust rewrite of Bun**

Claude Code now embeds a Rust port of Bun that reportedly starts 10% faster on Linux and has already reached millions of devices with little disruption. The uneventful rollout is a strong case study in using AI-assisted rewrites while treating compatibility and operational stability—not code-generation speed—as the real success criteria. [Read](https://simonwillison.net/2026/Jul/19/claude-code-in-bun-in-rust/)

**3. Secure Boot was bypassable for more than a decade**

Researchers found 11 vulnerable but still-trusted UEFI shims that could bypass Secure Boot on Windows and Linux, potentially enabling persistent bootkits; Microsoft revoked them only after disclosure. The failure highlights that signed software is only as trustworthy as the revocation pipeline behind it. [Read](https://arstechnica.com/security/2026/07/microsoft-secure-boot-has-been-broken-for-most-of-its-existence/)

**4. Slack brings immutable deployment practices to EC2 fleets**

Slack’s Shipyard replaces continuously mutated, long-lived instances with layered images, progressive metric-gated rollouts, automated rollback, fleet inventory, and regularly rotated hosts. It offers a useful modernization pattern for large workloads that still need VMs rather than containers. [Read](https://slack.engineering/shipyard-how-we-built-slacks-next-generation-ec2-platform/)

**5. Turso is building a Postgres-compatible database in Rust**

Turso plans to compile multiple SQL frontends—including a new Postgres-compatible one—onto a shared database virtual machine, aiming for async operation, embedded deployment, concurrent writes, and self-updating materialized views. If compatibility holds, the architecture could separate SQL ecosystems from decades-old execution engines much as LLVM separated languages from machine backends. [Read](https://turso.tech/blog/a-new-modern-version-of-postgres-in-rust)
