# Tech Digest — 2026-07-04

**1. Leaking YouTube creators' private videos**
A bug-bounty writeup shows how private YouTube creator videos could be exposed, a reminder that media platforms’ access-control edges are as critical as their serving architecture. For engineering leaders, it is a useful case study in authorization testing around preview, processing, and sharing workflows. [Read](https://javoriuski.com/post/youtube)

**2. Potential session/cache leakage between Claude Code workspace instances**
A reported Claude Code issue describes apparent cross-workspace or cross-account context leakage, including an agent suddenly referencing unrelated user state. Even if the root cause is still being investigated, it highlights why enterprise AI tools need strict tenant isolation, cache boundaries, and auditable session provenance. [Read](https://github.com/anthropics/claude-code/issues/74066)

**3. Better Models: Worse Tools**
Armin Ronacher argues that newer, stronger models can still regress in tool-calling behavior, making agentic workflows less reliable despite better raw reasoning. The takeaway is practical: production AI systems need evals for orchestration, tool-use contracts, and failure modes—not just benchmark scores. [Read](https://lucumr.pocoo.org/2026/7/4/better-models-worse-tools/)

**4. Why implementing ActivityPub is hard, and why it doesn't have to be**
This deep dive explains the messy reality behind federated protocols: fragmented signatures, JSON-LD variation, delivery ordering, SSRF risks, and platform-specific interoperability traps. It is a good distributed-systems lesson for any team building cross-tenant or federated infrastructure. [Read](https://hackers.pub/@fedify/2026/why-activitypub-is-hard)

**5. Fable created novel 4D splat format**
Fable’s `.splat4d` format targets streamable 4D Gaussian splats with tunable error bounds, large compression gains, HTTP Range support, and a WebGPU viewer. It matters because volumetric and spatial media will need CDN-friendly formats before they can become practical product surfaces. [Read](https://adamraudonis.github.io/splats4D/)
