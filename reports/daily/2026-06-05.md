# Tech Digest — 2026-06-05

**1. pg_durable: PostgreSQL in-database durable execution**
Microsoft open-sourced a Postgres extension for running durable workflows inside the database, aiming to make state, retries, and recovery transactional instead of split across queues and app code. For engineering leaders, it is a useful signal that “workflow engine + database” boundaries are being rethought around operational simplicity. [Read](https://github.com/microsoft/pg_durable)

**2. Gemma 4 QAT models: optimizing compression for mobile and laptop efficiency**
Google released quantization-aware trained Gemma 4 checkpoints that reduce memory needs while preserving more model quality than post-training quantization. This matters for teams trying to move AI features onto laptops, phones, and edge devices without paying server-side inference costs for every interaction. [Read](https://blog.google/innovation-and-ai/technology/developers-tools/quantization-aware-training-gemma-4/)

**3. Did Claude increase bugs in rsync?**
This analysis looks at whether AI-assisted changes in rsync correlated with more defects, using real project history rather than vibes. The practical takeaway is not “ban AI coding,” but that critical infrastructure needs review, tests, and maintainership discipline that assume generated code can be confidently wrong. [Read](https://alexispurslane.github.io/rsync-analysis/)

**4. Changing how we develop Ladybird**
Ladybird is tightening its contribution process as it moves toward shipping a real browser, including how it handles AI-generated or low-context changes. It is a good case study in protecting a complex systems codebase from review overload while still keeping open-source momentum. [Read](https://ladybird.org/posts/changing-how-we-develop-ladybird/)

**5. Redis 8.8: new array data structure, rate limiter, and performance improvements**
Redis 8.8 adds a native array data type, a window-counter rate limiter, stream NACK support, and performance work across core operations. For backend teams, this is a reminder that mature infrastructure keeps absorbing common application patterns, potentially simplifying hot-path services and shared platform code. [Read](https://redis.io/blog/announcing-redis-8-8/)
