# Tech Digest — 2026-05-22

**1. Project Glasswing: An Initial Update**
Anthropic says its AI-assisted security effort has found more than 10,000 high- or critical-severity vulnerabilities across important open-source software, shifting the bottleneck from discovery to verification, disclosure, and patching. For engineering leaders, this is a preview of security programs where AI increases triage and remediation load as much as it improves detection. [Read](https://www.anthropic.com/research/glasswing-initial-update)

**2. Megalodon: Mass GitHub Repo Backdooring via CI Workflows**
SafeDep reports a campaign that pushed thousands of malicious commits replacing GitHub Actions workflows with secret-exfiltration payloads, including backdoored npm releases. It is a sharp reminder that CI configuration is production attack surface and needs the same review, provenance checks, and monitoring as application code. [Read](https://safedep.io/megalodon-mass-github-repo-backdooring-ci-workflows)

**3. Google API keys keep working after you delete them long enough to be exploited**
Aikido found deleted Google API keys can continue authenticating for up to about 23 minutes after removal, creating a risky gap during incident response. Teams should treat key deletion as eventually consistent, rotate defensively, and design revocation playbooks around observed propagation delays. [Read](https://www.aikido.dev/blog/google-api-keys-deletion)

**4. Aggressive AI scrapers are making it kinda suck to run wikis**
Weird Gloop details how LLM-era scraper traffic can dwarf human wiki traffic, create outages, and force operators into costly bot-mitigation work. The broader lesson is that AI data demand is becoming an infrastructure externality for public web services, especially long-tail knowledge sites. [Read](https://weirdgloop.org/blog/clankers)

**5. Models.dev: open-source database of AI model specs, pricing, and capabilities**
Models.dev is an open database for comparing model context windows, pricing, modalities, and capabilities across providers. As model choice becomes a routing and cost-control problem, a maintained machine-readable catalog can help teams standardize evaluation and vendor abstraction. [Read](https://github.com/anomalyco/models.dev)
