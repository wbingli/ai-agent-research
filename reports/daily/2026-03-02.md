# AI Agent Research Daily Digest — 2026-03-02

## 🔥 Top Finds

### 1. git-memento — Record AI sessions per commit
- **Repo:** [mandel-macaque/memento](https://github.com/mandel-macaque/memento)
- **HN:** 388 points, 331 comments — huge discussion
- **What:** Git extension that attaches AI coding session transcripts as git notes on commits. Supports Codex and Claude Code. Includes `git memento commit`, `audit`, `share-notes`, and rebase-safe note carry-over.
- **Why interesting:** Addresses a real governance gap — if AI writes your code, the session should be auditable. Could become standard practice for teams using coding agents. The HN discussion "If AI writes code, should the session be part of the commit?" struck a nerve.

### 2. Manifest — Smart LLM routing for OpenClaw
- **Repo:** [mnfst/manifest](https://github.com/mnfst/manifest) — ⭐ 3,486 (43 stars today, trending)
- **What:** OpenClaw plugin that intercepts queries and routes them to the most cost-effective model using a 23-dimension scoring algorithm (<2ms overhead). Real-time cost dashboard, OTLP-native telemetry, soft/hard spending limits.
- **Why interesting:** Claims up to 70% cost reduction. Runs entirely local (no third-party data sharing). Simple install as OpenClaw plugin. Directly relevant for anyone running multi-model setups.

### 3. Omni — Open-source workplace AI search + chat
- **Repo:** [getomnico/omni](https://github.com/getomnico/omni) — Show HN, 103 points
- **What:** Self-hosted workplace search platform. Connects Google Drive/Gmail, Slack, Confluence, Jira. Uses Postgres (ParadeDB) for everything — BM25 full-text + pgvector semantic search. AI agent with sandboxed code execution. Core services in Rust + Python + SvelteKit.
- **Why interesting:** Single-database architecture (no Elasticsearch, no separate vector DB) is a refreshing design choice. Permission inheritance from source systems. The "Postgres for everything" approach is elegant and ops-friendly.

### 4. GitHub Enterprise AI Controls & Agent Control Plane (GA)
- **Link:** [GitHub Changelog (Feb 26)](https://github.blog/changelog/2026-02-26-enterprise-ai-controls-agent-control-plane-now-generally-available/)
- **What:** GitHub's enterprise governance layer for AI agents is now GA. Includes audit logs, session tracking, centralized policies, and API support for custom agent definitions via `.github-private/agents/*.md`.
- **Why interesting:** Signals that enterprise AI agent governance is becoming a first-class concern. The `.github-private/agents/*.md` convention for defining custom agents is worth noting — could become a de facto standard.

### 5. Khoj — AI second brain (trending again)
- **Repo:** [khoj-ai/khoj](https://github.com/khoj-ai/khoj) — trending on GitHub today
- **What:** Self-hostable personal AI that connects to your docs, the web, and any LLM. Custom agents with tunable personality/tools/knowledge bases. Automated research with smart notifications. Accessible from browser, Obsidian, Emacs, WhatsApp, phone.
- **Why interesting:** Keeps trending because it's one of the most complete open-source "personal AI" platforms. The custom agent creation and automated research features overlap with what we're building with OpenClaw but from a different angle.
