### 1. Karpathy's `autoresearch` — Autonomous AI Research on a Single GPU
- **Link:** https://github.com/karpathy/autoresearch
- **Stars:** Rising fast (185 points on HN front page)
- **What it does:** Give an AI agent a single GPU + a real LLM training setup (nanochat). The agent autonomously modifies `train.py`, trains for 5 minutes, evaluates val_bpb, keeps or discards changes, and repeats overnight. You wake up to a log of experiments and (hopefully) a better model.
- **How it works:** Three files — `prepare.py` (fixed data prep), `train.py` (agent-editable training code), `program.md` (human-written instructions for the agent). The agent iterates on architecture, hyperparameters, optimizer, batch size — everything in train.py. ~12 experiments/hour.
- **Why interesting:** This is "research as programming the researcher." You write `program.md` to define your research org, not Python. Karpathy frames this as the origin story of autonomous AI research — and the concept scales naturally to multi-agent research orgs. Directly relevant to anyone doing local LLM experimentation.

### 2. Agent Safehouse — macOS-Native Sandboxing for Local AI Agents
- **Link:** https://agent-safehouse.dev / https://github.com/eugene1g/agent-safehouse
- **Stars:** 685 points on HN (#1 trending)
- **What it does:** A single Bash script that provides kernel-level sandboxing for AI coding agents on macOS. Deny-first access model — agents can only touch explicitly granted directories.
- **How it works:** Uses macOS native sandboxing (likely `sandbox-exec`). Download one shell script, run `safehouse claude --dangerously-skip-permissions` and your agent is jailed. SSH keys, other repos, personal files — all denied at the kernel level. Can wrap any agent (Claude Code, Codex, etc.).
- **Why interesting:** Solves the "yolo mode" anxiety perfectly. With probabilistic LLMs, a 1% chance of `rm -rf ~` is a matter of when, not if. This makes running agents with full permissions actually safe. Dead simple setup — no build step, no dependencies. Should be standard practice for anyone running local agents.
