# Tech Digest — 2026-05-06

**1. Agents can now create accounts, buy domains, and deploy apps**
Cloudflare is showing agents that can provision accounts, pay with Stripe, register domains, and deploy projects. The shift is agents becoming controlled operators for real infrastructure, not just chat helpers. [Read](https://blog.cloudflare.com/agents-stripe-projects/)

**2. Faster Gemma inference with multi-token prediction**
Google uses small draft models to predict multiple future tokens, then has the main model verify them. This improves serving speed without changing the model’s output quality much. [Read](https://blog.google/innovation-and-ai/technology/developers-tools/multi-token-prediction-gemma-4/)

**3. Computer-use agents are far more expensive than structured APIs**
Browser/computer-control agents can be dramatically more expensive than direct API integrations. Use agents for messy workflows, but expose clean tools/APIs whenever you control the system. [Read](https://reflex.dev/blog/computer-use-is-45x-more-expensive-than-structured-apis/)

**4. Nix/Lix local privilege escalation**
The Nix/Lix ecosystem has a local privilege-escalation issue, with an integer-overflow writeup explaining the exploit path. Worth watching if you use Nix on developer laptops or CI, because build tooling often has broad trust. [Read](https://lix.systems/blog/2026-05-05-lix-unsigned-integer-overflow/)

**5. Formatting 25 million lines overnight**
Stripe’s rubyfmt story is about landing a huge mechanical codebase change safely. The useful lesson is how to batch, automate, and reduce merge pain when changing millions of lines. [Read](https://stripe.dev/blog/formatting-an-entire-25-million-line-codebase-overnight-the-rubyfmt-story)
