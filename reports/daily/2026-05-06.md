# Tech Digest — 2026-05-06

**1. Gemma 4 inference gets faster with multi-token prediction**
Google describes “draft” models that predict multiple future tokens at once, then let the main model verify them, delivering up to 3× faster Gemma 4 inference. The practical takeaway for AI platform leaders is that latency and serving cost can improve materially without changing the user-facing model API.

**2. Chrome is reportedly installing a 4 GB on-device AI model by default**
A privacy-focused analysis claims Chrome is silently downloading Gemini Nano onto users’ machines, raising consent, storage, bandwidth, and fleet-management concerns. Even if the policy debate evolves, engineering leaders should treat browser-bundled local models as a new operational surface: disk budgets, update controls, telemetry, and enterprise opt-outs now matter.

**3. Docker Compose can still run production workloads—if you close the gaps**
A pragmatic production checklist argues plain Docker Compose remains viable for small-to-medium systems when teams explicitly handle healing, cleanup, image pinning, socket security, backups, and updates. This is useful counterweight to defaulting to Kubernetes: complexity should be bought for concrete operational needs, not status.

**4. Cloudflare is making agents first-class cloud customers**
Cloudflare says agents can now create accounts, buy domains, start paid subscriptions, obtain API tokens, and deploy code with humans approving sensitive steps. This points toward a near-term platform shift where infrastructure onboarding flows need agent-safe identity, spending limits, audit trails, and revocable permissions.

**5. A Podman rootless-container exploit highlights copy/ownership footguns**
A security writeup on the “Copy Fail” exploit shows how container file-copy behavior and UID/GID mapping can create surprising privilege boundaries even in rootless setups. For infrastructure teams, it is a reminder that “rootless” reduces risk but does not eliminate the need to threat-model developer tooling and CI container workflows.
