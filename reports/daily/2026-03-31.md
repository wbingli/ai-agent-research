# AI Agent Research Daily Digest — 2026-03-31

## 🔥 Top Discoveries

### 1. Headroom — Context Compression Layer for AI Agents
- **Repo:** https://github.com/chopratejas/headroom
- **What:** A proxy/SDK that sits between your agent and the LLM provider, compressing tool outputs, DB reads, RAG results, and file reads before they hit the model. Claims 70-95% of agent context is boilerplate that can be stripped.
- **How:** Transparent proxy (`headroom proxy`), Python/TS SDK (`compress()`), or framework integrations (LangChain, LiteLLM, Agno). Has `headroom wrap claude` for zero-config wrapping of Claude Code, Codex, Aider, Cursor.
- **Why interesting:** Token cost is the #1 pain point for agent-heavy workflows. Instead of optimizing prompts one-by-one, Headroom generalizes compression as infrastructure. SharedContext API lets multi-agent systems reuse compressed context. Works with OpenClaw, Claude Code, Codex — basically anything. This is the kind of boring-but-essential middleware that actually saves money at scale.

### 2. Craft Agents — GUI-First Agent Workspace (Open Source)
- **Repo:** https://github.com/lukilabs/craft-agents-oss
- **Stars:** New (from craft.do team)
- **What:** A beautiful, document-centric UI for working with AI agents. Built on Claude Agent SDK + Pi SDK. Uses "Agent Native" software principles — the team builds Craft Agents entirely with Craft Agents (no code editor).
- **How:** Tell the agent "add Linear as a source" and it auto-discovers APIs/MCP servers, reads docs, configures credentials. Supports stdio MCP servers, OpenAPI specs, direct DB connections. Can import Claude Code skills.
- **Why interesting:** This is the anti-CLI thesis. While everyone builds more terminal tools, Craft Agents bets on a visual, document-centric workflow. The "bootstrapping" angle (building itself with itself) is compelling. Apache 2.0 licensed. Could be interesting for non-engineer team members who want agent power without CLI fluency.

### 3. Coasts — Containerized Dev Environments for Parallel Agents
- **Repo:** https://github.com/coast-guard/coasts
- **HN:** https://news.ycombinator.com/item?id=47575417 (84 points, trending)
- **What:** CLI + observability UI for running N isolated instances of your full dev environment on one machine. Uses git worktrees + Docker. Small `Coastfile` at repo root, or reuses existing docker-compose.yml.
- **How:** `coast checkout` binds canonical ports to one worktree at a time; dynamic ports let you peek into others. Agent-harness agnostic — works with Claude Code, Codex, Cursor, Conductor.
- **Why interesting:** Solves the real practical problem of running 3-5 coding agents in parallel without port conflicts, shared state corruption, or environment drift. Offline-first, no vendor lock-in. The observability dashboard for watching multiple agent worktrees is a nice touch. macOS-first but Linux works too.

### 4. jCodeMunch MCP — Token-Efficient Code Exploration
- **Repo:** https://github.com/jgravelle/jcodemunch-mcp
- **What:** MCP server that indexes codebases via tree-sitter AST parsing, letting agents retrieve exact functions/classes/methods with byte-level precision instead of brute-reading entire files.
- **How:** Index once, then agents query symbols, outlines, and context bundles. Claims 95%+ reduction in code-reading token usage.
- **Why interesting:** Directly addresses the "token incinerator" problem of agents opening whole files to find one function. Tree-sitter gives language-aware precision across many languages. Pairs well with Headroom (compress what you read) — different layers of the same efficiency stack.

### 5. Awesome Agent Skills — The Definitive Skills Directory
- **Repo:** https://github.com/VoltAgent/awesome-agent-skills
- **What:** Curated collection of 1000+ agent skills from official dev teams (Anthropic, Google, Vercel, Stripe, Cloudflare, Netlify, Trail of Bits, Sentry, Expo, Hugging Face, Figma, Microsoft...) plus community contributions.
- **Why interesting:** This is becoming the npm/PyPI of agent skills. Unlike bulk-generated repos, these are real skills from actual engineering teams. Compatible with Claude Code, Codex, Gemini CLI, Cursor, GitHub Copilot, OpenCode, Windsurf. Good reference for building OpenClaw skills too.

## 🔮 Trends

1. **Token efficiency is becoming infrastructure** — Headroom (compression proxy) + jCodeMunch (AST-based retrieval) represent a new layer in the agent stack focused purely on reducing token waste. Expect more "middleware for agents" projects.
2. **Agent workspace UIs diverging** — Craft Agents (GUI-first, document-centric) vs CLI tools (Claude Code, Codex). The market is splitting between engineer-facing and everyone-else-facing agent interfaces.
3. **Parallel agent isolation is solved** — Coasts (Docker + worktrees) joins yesterday's CLI-Anything in the "make agents work in the real world" category. The tooling around agents is maturing fast.
4. **Skills ecosystems consolidating** — VoltAgent's awesome-agent-skills has become the central directory. Skills are the new packages.
5. **Agent security scrutiny intensifying** — "Agents of Chaos" paper, "Copilot ad injection", and the "vulnerability research is cooked" thread all point to growing awareness that agent autonomy has real risks.
