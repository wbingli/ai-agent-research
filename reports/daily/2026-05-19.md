# Tech Digest — 2026-05-19

**1. Forge: Guardrails take an 8B model from 53% to 99% on agentic tasks**
Forge is a Python framework for self-hosted LLM tool-calling and multi-step workflows, with the headline claim that structured guardrails dramatically improve small-model agent reliability. For engineering leaders, the important takeaway is architectural: orchestration, validation, and recovery loops may matter as much as raw model size for production agents. [Read](https://github.com/antoinezambelli/forge)

**2. CISA admin leaked AWS GovCloud keys on GitHub**
Krebs reports that a CISA contractor exposed highly privileged AWS GovCloud credentials and internal system access through a public GitHub repository. It is a sharp reminder that secret scanning, least privilege, key rotation, and contractor repo governance are not optional controls—even for security agencies. [Read](https://krebsonsecurity.com/2026/05/cisa-admin-leaked-aws-govcloud-keys-on-github/)

**3. Cloudflare: Project Glasswing and security-focused frontier models**
Cloudflare tested Mythos and other security-oriented LLMs against live code in critical infrastructure and shared where they helped, where they failed, and what is needed before scaling this workflow. The useful signal is pragmatic: AI security review is becoming real, but needs tight human review, evaluation discipline, and integration into existing engineering systems. [Read](https://blog.cloudflare.com/cyber-frontier-models/)

**4. Gemini 3.5: frontier intelligence with action**
Google announced Gemini 3.5, positioning the new model line around stronger reasoning plus action-oriented capabilities for agents and multimodal workflows. For teams building AI products, this reinforces the direction of travel: model roadmaps are converging on tool use, task execution, and richer context rather than chat-only assistants. [Read](https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-5/)

**5. Turso used Quint to find 10+ bugs in SQLite**
Turso modeled parts of the SQLite C API with Quint, generated traces, and ran them against SQLite while hardening its own database stack. This is a strong example of formal methods moving from academic niche to practical reliability tooling for infrastructure teams with high correctness requirements. [Read](https://turso.tech/blog/how-we-used-quint-to-find-over-10-bugs-in-sqlite)
