# AI Agent Research Daily Digest — 2026-03-16

## 🔥 Top Discoveries

### 1. Karpathy's AutoResearch — Autonomous AI Experiment Loops (37.7K ⭐)
- **GitHub:** [karpathy/autoresearch](https://github.com/karpathy/autoresearch)
- **Stars:** 37,749 (created Mar 6 — 37K stars in 10 days!)
- **What:** A 630-line Python script that lets AI agents autonomously design, run, and analyze ML experiments on a single GPU overnight. Originally focused on nanochat training.
- **How:** An agentic loop where an LLM proposes experiment variations, generates training configs, runs them on GPU, collects metrics, analyzes results, and proposes next experiments — all without human input. Ran 50+ experiments in one night in initial demo.
- **Why interesting:** This is the "vibe research" moment. The design pattern (autonomous propose→execute→analyze→iterate loop) generalizes far beyond ML training — could apply to any domain with programmatic evaluation. Karpathy's endorsement gives it massive credibility. The simplicity (630 lines!) makes it immediately forkable for custom research domains.

### 2. Hindsight — Agent Memory That Learns Like Human Memory (4K ⭐, trending)
- **GitHub:** [vectorize-io/hindsight](https://github.com/vectorize-io/hindsight)
- **Stars:** 4,068 (trending since Mar 15)
- **Docs:** [hindsight.vectorize.io](https://hindsight.vectorize.io/)
- **What:** A memory system for AI agents that mimics human memory pathways — separating "world facts" from "experiences," with entities, relationships, and learned abstractions.
- **How:** Memories are stored in "banks." When added, they're routed into either world facts or experiences pathways, then decomposed into entities, relationships, and higher-level abstractions. Supports MCP integration and has a `npx skills add` installer for coding agents.
- **Why interesting:** Directly relevant to how we manage agent memory (MEMORY.md, daily files). Hindsight formalizes what many agent systems do ad-hoc: separating declarative knowledge from episodic memory. The dual-pathway architecture (facts vs experiences) maps well to how humans actually remember. Could be a building block for long-running agents that genuinely learn.

### 3. Alibaba's PageAgent — In-Page GUI Agent for Natural Language Web Control (9.3K ⭐)
- **GitHub:** [alibaba/page-agent](https://github.com/alibaba/page-agent)
- **Stars:** 9,289
- **What:** A JavaScript library that embeds a GUI agent directly into web pages. Users control web interfaces with natural language commands — no browser extension, no external tool, just a JS import.
- **How:** Runs in-page (not externally like Playwright/Puppeteer), so it has direct DOM access. Uses vision models + DOM analysis to understand the page and execute natural language instructions. Minimal integration — add a script tag and go.
- **Why interesting:** Different approach from MidScene.js (yesterday) and BrowseUse. Instead of controlling the browser from outside, PageAgent lives *inside* the page. This means: (1) no CORS issues, (2) access to shadow DOM, (3) works with any web app without browser extensions. For web app developers wanting to add AI assistance to their own products, this is more practical than external automation.

### 4. Superpowers — Agentic Skills Framework Hits 87K Stars
- **GitHub:** [obra/superpowers](https://github.com/obra/superpowers)
- **Stars:** 87,747
- **What:** A composable "skills" framework and software development methodology for coding agents (Claude Code, Copilot, Cursor, Gemini CLI, etc.). Defines how agents should approach brainstorming, coding, reviewing, and testing.
- **How:** Skills are structured markdown files that override/augment default agent behavior. Includes a TDD-inspired approach: write test scenarios, observe baseline agent behavior, write skill docs to correct it, verify compliance. Skills cover brainstorming, code review, writing, and more.
- **Why interesting:** At 87K stars, this is arguably the most popular coding agent methodology. The insight that agent behavior is best shaped through structured documentation (not code) is powerful. The "write tests for agent behavior" paradigm is novel — treating prompt engineering like software testing.

### 5. DimOS — Agentic OS for Physical Robots (1.4K ⭐, rising)
- **GitHub:** [dimensionalOS/dimos](https://github.com/dimensionalOS/dimos)
- **Stars:** 1,445 (freshly trending Mar 16)
- **What:** An "agentic operating system" for controlling physical hardware — humanoids, quadrupeds, drones — using natural language. Build multi-agent systems that work with cameras, lidar, and actuators.
- **How:** Provides a framework to "vibecode" robots: describe desired behaviors in natural language, and the system translates to hardware commands. Supports multi-agent coordination across different physical platforms.
- **Why interesting:** The "agents meet physical world" frontier. While most agent frameworks target software tasks, DimOS bridges to hardware. Early stage (1.4K stars) but the vision of natural-language-programmable robots via agentic frameworks is compelling. Worth watching as embodied AI accelerates.

## 📊 Trends & Observations

- **Autonomous experiment loops are here:** Karpathy's autoresearch isn't just a tool — it's a template. Expect forks for drug discovery, materials science, financial modeling, and any domain with programmatic evaluation. The "agentic research" pattern will be huge in 2026.
- **Agent memory is getting serious:** Hindsight's dual-pathway architecture (facts vs experiences) signals that the field is moving beyond naive RAG for agent memory. Human-inspired memory architectures are the next frontier.
- **In-page vs external browser agents:** Two competing paradigms emerging — external automation (Playwright/BrowseUse/MidScene) vs in-page agents (PageAgent). In-page has advantages for first-party web apps; external wins for cross-site automation.
- **Skills > Code for agent behavior:** Superpowers at 87K stars proves that the best way to shape agent behavior is through structured documentation, not code. The "TDD for prompts" pattern is gaining traction.
