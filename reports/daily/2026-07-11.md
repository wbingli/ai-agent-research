# Tech Digest — 2026-07-11

**1. Mesh LLM: distributed AI computing on iroh**
Mesh LLM pools GPUs across multiple machines behind an OpenAI-compatible API, using iroh networking to make spare local or edge accelerators act like one inference cluster. This is a useful pattern for teams trying to reduce dependence on centralized GPU clouds while keeping developer ergonomics simple. [Read](https://www.iroh.computer/blog/mesh-llm)

**2. How ClickHouse scaled PgBouncer to 4x throughput**
ClickHouse describes how its Managed Postgres team worked around PgBouncer’s single-threaded ceiling by running peered processes with `SO_REUSEPORT`, spreading connection pooling across CPU cores. It is a practical reminder that mature infrastructure often scales through careful OS-level composition, not wholesale rewrites. [Read](https://clickhouse.com/blog/pgbouncer-clickhouse-managed-postgres)

**3. How GitHub gave every repository a durable owner**
GitHub explains how it assigned validated ownership across more than 14,000 internal repositories in under 45 days. For engineering leaders, the important lesson is that security and operational accountability need explicit ownership data, automation, and enforcement—not just tribal knowledge. [Read](https://github.blog/security/application-security/how-github-gave-every-repository-a-durable-owner/)

**4. Apple exploring larger on-device AI models for iPhones**
Apple is reportedly exploring PrismML techniques to run much larger compressed language models directly on iPhones, including a 27B-parameter Qwen variant. If this direction holds, more AI product architecture may shift toward private, low-latency edge inference instead of defaulting every interaction to the cloud. [Read](https://www.macrumors.com/2026/07/09/apple-prismml-larger-on-device-ai-models/)

**5. Xiaomi’s full-pipeline inference optimization for MiMo-V2.5**
Xiaomi published an engineering writeup on KV-cache management, tiered caching, SWA-aware prefix cache trees, scheduling, and prefill/decode pipeline optimization for multimodal inference. It is a dense but valuable look at the systems work now required to make large models economically serveable. [Read](https://mimo.xiaomi.com/blog/mimo-v2-5-inference)
