# Tech Digest — 2026-06-15

**1. A backdoor in a LinkedIn job offer**
A real-world social-engineering writeup shows how a fake recruiting flow can become a malware delivery path. It is a useful reminder that developer endpoints and hiring workflows are now part of the security perimeter. [Read](https://roman.pt/posts/linkedin-backdoor/)

**2. Iroh 1.0**
Iroh reached 1.0 with a “dial keys, not IPs” model for building peer-to-peer and distributed apps without forcing teams to manage addresses, NAT traversal, and connectivity plumbing directly. For infrastructure leaders, it points toward simpler primitives for edge, sync, and local-first systems. [Read](https://www.iroh.computer/blog/v1)

**3. North Mini Code: Cohere’s agentic coding model for developers**
Cohere introduced an open-source 30B MoE coding model aimed at agentic software development with modest hardware requirements. The important signal is continued pressure toward sovereign, self-hostable coding assistants rather than fully cloud-dependent developer tooling. [Read](https://cohere.com/blog/north-mini-code)

**4. The only scalable delete in Postgres is DROP TABLE**
PlanetScale argues that large deletes create more database work instead of reclaiming it, and recommends designing retention boundaries so deletion becomes partition drops or truncation. This is a practical architecture lesson for any team operating high-volume Postgres workloads. [Read](https://planetscale.com/blog/the-only-scalable-delete)

**5. TimescaleDB compression: Hypercore and columnar storage in PostgreSQL**
This deep dive explains how TimescaleDB’s Hypercore uses columnar storage and segment/order choices to reach high compression ratios for time-series data. It matters for teams trying to keep observability, IoT, or event data queryable without letting storage costs explode. [Read](https://roszigit.com/en/blog/timescaledb-compression-hypercore)
