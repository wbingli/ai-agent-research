# Tech Digest — 2026-06-30

**1. Claude Sonnet 5**
Anthropic’s new Sonnet release is positioned as its most agentic mid-tier model, with stronger coding and professional-work capabilities. For engineering leaders, the signal is that “default” coding assistants are continuing to move toward autonomous, multi-step execution rather than autocomplete. [Read](https://www.anthropic.com/news/claude-sonnet-5)

**2. “It’s Hard to Eval” Is a Product Smell**
Hamel Husain argues that AI products should be designed so outputs are easy to inspect, score, and debug—not treated as mysterious blobs. This is a useful management lens: eval infrastructure is not just QA, it shapes product architecture and velocity. [Read](https://hamel.dev/blog/posts/eval-smell/)

**3. I Ported Kubernetes to the Browser**
Ngrok describes running Kubernetes in a browser after generating nearly 100,000 lines of code with LLM assistance. Beyond the stunt, it is a concrete example of how AI-assisted development plus WebAssembly-style constraints can make complex infrastructure more portable and demoable. [Read](https://ngrok.com/blog/i-ported-kubernetes-to-the-browser)

**4. Qwen 3.6 27B Is the Sweet Spot for Local Development**
Quesma makes the case that Qwen 3.6 27B is now strong enough for practical local coding on MacBooks or RTX-class machines via llama.cpp and OpenCode. If true in your stack, this points to a lower-cost path for private, low-latency developer agents without routing every prompt to a hosted model. [Read](https://quesma.com/blog/qwen-36-is-awesome/)

**5. Soatok’s Informal Guide to Threat Models**
Soatok gives a pragmatic walkthrough of threat modeling, especially around privacy, cryptography, endpoint compromise, and policy debates like age verification. It is a reminder that security reviews are only useful when teams state the attacker, assets, and assumptions explicitly. [Read](https://soatok.blog/2026/06/30/soatoks-informal-guide-to-threat-models/)
