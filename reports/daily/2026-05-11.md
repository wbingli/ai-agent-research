# Tech Digest — 2026-05-11

**1. TanStack npm supply-chain compromise postmortem**
TanStack details an attack that chained `pull_request_target`, GitHub Actions cache poisoning, and OIDC token extraction to publish malicious versions across 42 npm packages. The lesson for engineering leaders: CI trust boundaries and package-publish credentials need the same threat modeling as production infra. [Read](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem)

**2. Mythos finds a curl vulnerability**
Daniel Stenberg walks through Anthropic’s AI-assisted curl vulnerability report and why “AI found a bug” still translated into ordinary maintainer triage, validation, and disclosure work. It’s a useful reality check on AI security tooling: the leverage is real, but the operational burden shifts to project owners. [Read](https://daniel.haxx.se/blog/2026/05/11/mythos-finds-a-curl-vulnerability/)

**3. Introducing the Claude Platform on AWS**
Anthropic’s Claude Platform is now generally available on AWS, bringing Claude’s developer platform features under AWS authentication, billing, and committed-spend retirement. For enterprises, this lowers procurement and governance friction around deploying agentic workflows in existing cloud environments. [Read](https://claude.com/blog/claude-platform-on-aws)

**4. Interaction Models: A scalable approach to human-AI collaboration**
Thinking Machines argues that AI products need to move beyond turn-based chat into richer interaction models that natively handle realtime, multimodal collaboration across text, audio, video, and shared state. This is a product-architecture signal: the next frontier is interface and workflow design, not just model quality. [Read](https://thinkingmachines.ai/blog/interaction-models/)

**5. CUDA-oxide: Nvidia’s Rust-to-CUDA compiler**
Nvidia’s CUDA-oxide project explores compiling Rust to CUDA, aiming to bring Rust’s safety and tooling to GPU programming without abandoning the CUDA ecosystem. If it matures, it could make accelerator codebases easier to maintain and safer for teams building ML systems and high-performance services. [Read](https://nvlabs.github.io/cuda-oxide/index.html)
