# AI Agent Research Daily Digest — 2026-03-30

## 🔥 Top Discoveries

### 1. CLI-Anything — Making ALL Software Agent-Native
- **Repo:** https://github.com/HKUDS/CLI-Anything
- **Stars:** 21,000+ (March 2026 release — explosive growth)
- **What:** Bridges the gap between GUI desktop software and AI agents by auto-generating CLI wrappers. One command turns any application into an agent-accessible tool for Claude Code, Cursor, Codex, OpenClaw, etc.
- **Why interesting:** This is a fundamental infrastructure play. Instead of building MCP servers one-by-one for every app, CLI-Anything generalizes the problem. Community-built CLIs are shared via a CLI-Hub. Solves the "last mile" problem of agent<>software integration at scale.
- **From:** HKU Data Science Lab (HKUDS) — same team behind OpenSpace and nanobot.

### 2. OpenSpace — Self-Evolving Agent Skills Engine
- **Repo:** https://github.com/HKUDS/OpenSpace
- **Stars:** 1,500+
- **What:** An MCP-based skill evolution engine. Skills auto-fix when they break, auto-improve from successful patterns, and auto-learn from actual usage. Agents share evolved skills across a community cloud, creating network effects.
- **Why interesting:** Moves beyond static SKILL.md files to living, self-improving skills. Claims 46% fewer tokens through pattern reuse. The "shared brain" concept — one agent's improvement upgrades all agents — is the kind of emergent behavior that compounds. Works with OpenClaw, Claude Code, Codex, Cursor.
- **From:** HKUDS (same lab as CLI-Anything)

### 3. Claude Peers MCP — Multi-Session Agent Coordination
- **Repo:** https://github.com/louislva/claude-peers-mcp
- **Stars:** ~500
- **What:** Lets multiple Claude Code instances on the same machine discover each other and communicate in real-time. A lightweight broker daemon (localhost:7899, SQLite) enables peer discovery, messaging, and status sharing.
- **Why interesting:** Solves a real pain point when running 3-5 Claude Code sessions across different projects. Agents can ask each other questions, coordinate work, avoid conflicts. Uses Claude's channel push for instant message delivery. Simple architecture — could be extended to cross-machine coordination.

### 4. Awesome AutoResearch — The AutoResearch Ecosystem Explodes
- **Repo:** https://github.com/alvinunreal/awesome-autoresearch
- **What:** Curated index of the rapidly growing ecosystem around Karpathy's AutoResearch (42K+ stars). Notable descendants:
  - **codex-autoresearch** — Codex-native port with resume support and parallel experiments
  - **gemini-autoresearch** — Gemini CLI port with Google Search grounding for live verification
  - **autocontext** (greyhaven-ai) — Closed-loop control plane with evaluation, persistent knowledge, and distillation to cheaper local runtimes
  - **goal-md** — Generalizes the autoresearch pattern to any measurable goal
- **Why interesting:** AutoResearch has become a *pattern*, not just a repo. The overnight autonomous improvement loop is being applied to software optimization, security, debugging, and beyond. The Gemini port using search grounding for verification is particularly clever.

### 5. Agent Shield — OS-Level Security for AI Coding Tools
- **Source:** [HN discussion](https://news.ycombinator.com/item?id=47498251)
- **What:** Background daemon that monitors AI coding tools (Claude Code, Cursor, Cline, Copilot, Codex) at the OS level using macOS FSEvents, lsof, and subprocess tree polling. All data stays in local SQLite.
- **Why interesting:** Security for AI agents is becoming critical. RSAC 2026 featured talks about AI coding tools as attack vectors (Check Point). Sysdig launched runtime security for coding agents. Cisco released agent security scanning tools. Agent Shield takes the indie approach — lightweight, local-first, privacy-preserving.
