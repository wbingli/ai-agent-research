# AI Agent Research Daily Digest — 2026-03-29 (Saturday)

## Top Discoveries

### 1. 🧠 lat.md — Agent Lattice: Knowledge Graph for Codebases in Markdown
- **Link:** https://github.com/1st1/lat.md
- **Stars:** ~54 (trending on HN today)
- **What:** A structured alternative to monolithic AGENTS.md/CLAUDE.md files. Creates a `lat.md/` directory of interconnected markdown files using `[[wiki links]]`, with source code back-references via `// @lat:` comments.
- **How:** CLI tool (`npm install -g lat.md`) with `lat check` (validates all links), `lat locate` (fuzzy/semantic search), and `lat refs` (find code implementing a concept). Supports exact, fuzzy, and vector search.
- **Why interesting:** Directly addresses the scaling problem we face with AGENTS.md. As codebases grow, a flat context file breaks down. This brings Obsidian-style linked knowledge to agent workflows. Could be very useful for our larger repos.

### 2. ⚡ flash-moe — Run 397B Parameter MoE Models on a MacBook
- **Link:** https://github.com/danveloper/flash-moe
- **Stars:** 1,847 (created Mar 18, 393 HN points)
- **What:** Pure C/Metal inference engine that runs Qwen3.5-397B-A17B on a MacBook Pro (48GB RAM) at 4.4 tok/s with working tool calling.
- **How:** SSD expert streaming (only loads K=4 active experts per layer from NVMe on demand), FMA-optimized Metal dequant kernels, hand-written Metal compute shaders. No Python, no frameworks. 209GB model streams from disk through custom Metal pipeline.
- **Why interesting:** Built by a CVS Health VP of AI in 24 hours (with AI assistance). Proves that massive MoE models are laptop-viable with clever engineering. The "Trust the OS" page cache approach is elegantly simple. This could change the local-inference game.

### 3. 💬 claude-peers-mcp — Real-Time Messaging Between Claude Code Instances
- **Link:** https://github.com/louislva/claude-peers-mcp
- **Stars:** 1,109 (created Mar 21)
- **What:** MCP server that lets multiple Claude Code sessions discover each other and exchange messages in real-time.
- **How:** Broker daemon on localhost:7899 with SQLite. Each Claude Code session registers via MCP, can list peers, send messages, and set status summaries. Messages delivered instantly via claude/channel protocol.
- **Why interesting:** Enables multi-agent collaboration across terminal windows — e.g., a writer agent and reviewer agent negotiating directly. This is essentially inter-process communication for AI agents. Very relevant to our OpenClaw multi-session setup.

### 4. 📚 codebase-to-course — Turn Any Repo into an Interactive HTML Course
- **Link:** https://github.com/zarazhangrui/codebase-to-course
- **Stars:** 1,055 (created Mar 22)
- **What:** A Claude Code skill that generates a self-contained single-page HTML course from any codebase — with scroll-based navigation, animated visualizations, code↔plain-English translations, and interactive quizzes.
- **How:** Claude Code skill (drop into `~/.claude/skills/`). Just say "Turn this codebase into an interactive course." Generates everything as one HTML file, no dependencies.
- **Why interesting:** Inverts CS education: build first, understand later. Targeted at "vibe coders" who build with AI but want to understand what they built. Great for onboarding onto unfamiliar codebases.

### 5. 🕸️ Miasma — AI Scraper Poison Trap
- **Link:** https://github.com/austin-weeks/miasma
- **Stars:** Trending on HN (110 points, front page today)
- **What:** A fast Rust server that traps AI web scrapers in an endless loop of poisoned training data and self-referential links.
- **How:** Hidden honeypot links on your site route scrapers to Miasma, which serves infinite poisoned pages with self-referencing links. Minimal memory footprint (~50-60MB for 50 concurrent connections).
- **Why interesting:** The anti-AI-scraping movement is growing. Creative adversarial approach — instead of blocking scrapers (whack-a-mole), it wastes their resources and poisons their data. The "endless buffet of slop for the slop machines" framing is chef's kiss.
