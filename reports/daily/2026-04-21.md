### 1. [Critical Anthropic MCP Design Vulnerability Enables RCE](https://www.ox.security/blog/the-mother-of-all-ai-supply-chains-critical-systemic-vulnerability-at-the-core-of-the-mcp/)
**Source:** Hacker News
**Summary:** OX Security disclosed a systemic "by-design" flaw in Anthropic's official MCP SDKs (Python, TypeScript, Java, Rust). The STDIO interface launches the target command even when the process fails to start correctly, so a malicious command still executes while the caller just sees an error. Estimated blast radius: 150M+ downloads, 7k+ public servers, up to 200k vulnerable instances. Anthropic classified the behavior as "expected."
**Why it matters:** Anyone wiring MCP servers into agent pipelines (including ad-tech agents that call internal tools) needs to audit their SDK usage now. This is the same class of supply-chain risk as npm post-install scripts, but in the AI tool-calling layer.

### 2. [Claude Code Project-File RCE & API Token Exfiltration (CVE-2026-21852)](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/)
**Source:** Hacker News
**Summary:** Check Point Research details how crafted Claude Code project files can trigger RCE and leak API tokens on developer machines. Pairs naturally with the MCP disclosure above — both are "the agent surface is now the attack surface."
**Why it matters:** If any team is running Claude Code against vendor-provided or user-uploaded repos (think: any pipeline that ingests external creative/bid-request data), the threat model just expanded.

### 3. [Claude Token Counter, Now with Model Comparisons](https://simonwillison.net/2026/Apr/20/claude-token-counts/)
**Source:** Simon Willison's weblog (Apr 20)
**Summary:** Simon upgraded his token counter to compare Opus 4.7, Opus 4.6, Sonnet 4.6, and Haiku 4.5. Opus 4.7 ships a new tokenizer — the same input maps to ~1.0–1.35× more tokens, and 1.46× on the Opus 4.7 system prompt itself. Per-token pricing is unchanged at $5/$25 per M, so effective cost quietly rose.
**Why it matters:** At ad-tech scale, a silent 35–46% token inflation is a huge line-item change. Anyone forecasting LLM spend on Opus 4.7 needs to re-baseline now rather than after the invoice.

### 4. [Qwen3.6-35B-A3B on My Laptop Drew a Better Pelican than Claude Opus 4.7](https://simonwillison.net/2026/Apr/16/qwen-beats-opus/)
**Source:** Simon Willison's weblog (Apr 16) / Hacker News
**Summary:** A 21GB quantized Qwen3.6-35B-A3B (35B total params, ~3B active per token) running locally on an M5 MacBook via LM Studio beat Opus 4.7 on Simon's pelican-on-a-bicycle SVG benchmark, and the flamingo-on-unicycle follow-up. Reported 73.4% SWE-Bench.
**Why it matters:** Open-weight MoE models at this quality unlock on-prem or edge agent deployments. For ad-tech, that means you can put the LLM layer next to the feature store / bid stream instead of round-tripping to a vendor API.
