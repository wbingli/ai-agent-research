# AI Agent Research Digest — April 1, 2026

## Top Discoveries

### 1. 🦀 OpenFang — Agent Operating System (Rust)
- **Repo:** [RightNow-AI/openfang](https://github.com/RightNow-AI/openfang) | ⭐ 16,079
- **What:** A full "Agent OS" written in Rust — 137K LOC, 14 crates, single binary. Positions itself as an infrastructure layer beneath agent frameworks, handling scheduling, permissions, inter-agent communication, and resource management.
- **Why interesting:** Pure Rust, battle-tested (1,767+ tests, zero clippy warnings). Takes the "OS for agents" metaphor literally — agents get process-like isolation, IPC, and a HAND.toml config (IaC-style). Feature-complete at v0.5.2. Being compared to OpenClaw but at a lower abstraction level.
- **Novel angle:** Infrastructure-as-Code config for agent topology. DevOps-native thinking applied to agent orchestration.

### 2. 🧠 Hermes Agent — Self-Improving Agent by Nous Research
- **Repo:** [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent) | ⭐ 21,190
- **What:** An autonomous agent that lives on your server, remembers what it learns, and gets more capable the longer it runs. Built by Nous Research (the Hermes model team).
- **Why interesting:** "The agent that grows with you" — it has persistent memory, learns from interactions, and self-improves. Uses their own Hermes models but also supports Claude/GPT. Has a Paperclip adapter for running as a managed "employee" in multi-agent companies. Comment-driven wake system for GitHub issues.
- **Novel angle:** Self-improvement loop + persistent server-side agent. Not just a CLI tool — it's a daemon that accumulates capability. Familiar concept to us (OpenClaw does similar), but from the model-maker's perspective.

### 3. 🤝 OpenAgents — Agent Networks for Open Collaboration
- **Repo:** [openagents-org/openagents](https://github.com/openagents-org/openagents) | ⭐ 2,210
- **What:** "Slack for agents" — a workspace where multiple AI agents can be pulled into conversation threads, sharing files, browser, and context. No glue code needed between agents.
- **Why interesting:** Focuses on multi-agent collaboration UX rather than just orchestration plumbing. Agents share a persistent workspace. Apache 2.0. The framing as "Slack for agents" is compelling — it's the social layer for agent-to-agent work.
- **Novel angle:** Workspace-centric rather than pipeline-centric multi-agent design.

### 4. ⚓ Harbor — Agent Evaluation in Container Environments
- **Repo:** [harbor-framework/harbor](https://github.com/harbor-framework/harbor) | ⭐ 1,200
- **What:** Framework for running agent evaluations and creating RL environments in containers. Standardized benchmarking for agents.
- **Why interesting:** As agents proliferate, evaluation becomes critical. Harbor provides reproducible, containerized environments for testing agent capabilities. Think "CI for agents."
- **Novel angle:** RL environments + eval in one framework. Useful for anyone building agents who needs to measure if they're actually getting better.

### 5. 📝 Universal CLAUDE.md — 63% Token Reduction
- **Repo:** [drona23/claude-token-efficient](https://github.com/drona23/claude-token-efficient) | ⭐ 1,874
- **What:** A drop-in CLAUDE.md file that cuts Claude Code output tokens by ~63% with no code changes. Optimized system prompting.
- **Why interesting:** Hit HN front page (463 points). Practical and immediately useful. Shows that prompt engineering at the system level still has massive gains. The "universal" framing means it works across projects.
- **Novel angle:** Meta-optimization — optimizing the agent's own instructions rather than the agent's code.
