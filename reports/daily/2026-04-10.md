# AI Agent Research Daily Digest — April 10, 2026

## Top Discoveries

### 1. 🧠 MemPalace — "Highest-scoring AI memory system ever benchmarked"
- **Repo:** [milla-jovovich/mempalace](https://github.com/milla-jovovich/mempalace) — 39.1k stars
- **What:** A free, open memory layer for LLM agents claiming top benchmark scores on agent memory tasks. Explosive growth (created this month).
- **Why interesting:** Memory is the current bottleneck for long-horizon agents. If the benchmarks hold up, this would be a drop-in upgrade for any harness that struggles with context across sessions. The viral trajectory alone makes it worth reading the architecture (teardown in `awesome-ai-anatomy` below also covers it).

### 2. 📖 LLM Wiki — Karpathy's "read before you answer" RAG replacement
- **Repos:** [nashsu/llm_wiki](https://github.com/nashsu/llm_wiki) (510⭐) and [lucasastorian/llmwiki](https://github.com/lucasastorian/llmwiki) (196⭐)
- **What:** Instead of classic RAG (chunk → embed → retrieve → stuff into context), these tools have an LLM *build and maintain* a hyperlinked wiki from your documents upfront. At query time the agent navigates the wiki like a human would.
- **How it works:** Documents are ingested, the LLM writes encyclopedic entries with cross-links, then a reader agent follows links to synthesize answers. Two independent open implementations appearing the same week suggests Karpathy's LLM Wiki talk is catching fire.
- **Why interesting:** This is genuinely a new pattern — structured knowledge distillation instead of semantic search. Likely better for domains where relationships matter (codebases, research corpora) and worse for fast-moving data. Worth trying against our own docs.

### 3. 🔬 Research-Driven Agents — "When an agent reads before it codes"
- **HN:** [197 points, 47706141](https://news.ycombinator.com/item?id=47706141)
- **What:** Essay + technique: force the coding agent to read relevant code and docs *before* generating a plan, not during. Turns the agent into a "researcher first, coder second."
- **Why interesting:** Matches what the best human engineers do. Aligns with LLM Wiki (#2) — both are betting that preprocessing/reading is higher leverage than clever retrieval at generation time. Practical pattern to try in Claude Code / Codex prompts today.

### 4. 🔍 awesome-ai-anatomy — Source-code teardowns of 15 coding agents
- **Repo:** [NeuZhou/awesome-ai-anatomy](https://github.com/NeuZhou/awesome-ai-anatomy) — 104⭐
- **What:** Architecture diagrams and annotated teardowns of Claude Code, Codex CLI, Cline, OpenHands, Goose, Dify, MemPalace, oh-my-codex, and 7 more.
- **Why interesting:** Single best resource if you want to understand how production agent harnesses actually work under the hood — prompt structure, tool loops, context management. Cuts weeks off research time.

### 5. 🎓 mini-coding-agent — Sebastian Raschka's minimal harness
- **Repo:** [rasbt/mini-coding-agent](https://github.com/rasbt/mini-coding-agent) — 558⭐
- **What:** Minimal, readable Python implementation of a coding agent harness. Explains the core components (planner, tool loop, file ops, shell) in as little code as possible.
- **Why interesting:** Raschka's pedagogy is excellent (LLMs-from-scratch author). This is the companion piece to `awesome-ai-anatomy` — read the teardowns, then build it yourself in ~200 lines. Great onboarding for anyone building on Claude Agent SDK.

### TUI-use — Agents that drive interactive terminal programs
- [HN 52 pts](https://news.ycombinator.com/item?id=47692661). Lets LLMs interact with curses/full-screen TUIs (think `htop`, `vim`, REPL debuggers) that normal shell-exec tools can't handle. Similar spirit to Browser Use but for the terminal. Potentially huge for debugging and ops automation.

### GLM-5.1 claims Opus 4.6 parity at 1/3 cost
- [HN 22 pts](https://news.ycombinator.com/item?id=47682384). Zhipu's GLM-5.1 benchmarks near Opus 4.6 on agentic tasks at roughly a third the price. If the real-world agentic behavior holds, that's a meaningful cost shift for long-horizon runs.

### browser39 — Headless browser purpose-built for agents
- [alejandroqh/browser39](https://github.com/alejandroqh/browser39) (36⭐). Single-binary headless browser that converts pages to token-optimized markdown locally. No Chromium dep, no cloud fees. Interesting lightweight alternative to Playwright-based browser agents.
