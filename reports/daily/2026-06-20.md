# Tech Digest — 2026-06-20

**1. Temporary Cloudflare Accounts for AI agents**
Cloudflare is letting agents create temporary Workers accounts and deploy live code without a human signup flow. This is a practical step toward agent-native dev infrastructure, with clear implications for sandboxing, quotas, abuse controls, and CI-style ephemeral environments. [Read](https://blog.cloudflare.com/temporary-accounts/)

**2. Bun has an open PR adding shared-memory threads to JavaScriptCore**
Bun is experimenting with shared-memory threading inside JavaScriptCore, a runtime-level change that could make JS concurrency more capable for server workloads. Engineering leaders should watch this because runtime primitives often reshape what teams can build without moving hot paths out of JavaScript. [Read](https://github.com/oven-sh/WebKit/pull/249)

**3. Linux finally eliminates the strncpy API after six years of work**
The Linux kernel has removed `strncpy` after a multi-year, 360+ patch cleanup to reduce an old source of unsafe string handling. It is a strong example of security work as sustained engineering hygiene, not just incident response. [Read](https://www.phoronix.com/news/Linux-7.2-Drops-strncpy)

**4. PostgresBench: A Reproducible Benchmark for Postgres Services**
ClickHouse published PostgresBench, an open benchmark for comparing managed Postgres services under reproducible conditions. For platform teams, the useful part is less the vendor ranking and more the push toward transparent workload definitions, repeatable methodology, and evidence-based database buying decisions. [Read](https://clickhouse.com/blog/postgresbench)

**5. Inference cost at scale with napkin math**
This writeup breaks LLM inference economics into simple capacity, latency, and utilization estimates instead of hand-wavy per-token pricing. It is useful for leaders deciding when AI features are product margin, infra planning, or just an expensive demo. [Read](https://injuly.in/blog/napkin-inference-cost/index.html)
