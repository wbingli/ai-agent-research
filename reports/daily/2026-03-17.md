# AI Agent Research Daily Digest — 2026-03-17

## 🔥 Top Discoveries

### 1. Leanstral — Mistral's Open-Source Formal Proof Agent
- **Link:** https://mistral.ai (announced March 16)
- **Stars:** Trending on HN (#9, 636 points)
- **What:** First open-source agent built specifically for **formal verification** using Lean 4. Only 6B active parameters but beats models 100x its size (Qwen3.5 397B, Kimi-K2.5 1T, GLM5 744B) on formal proof benchmarks.
- **How:** Specialized for proof engineering workflows — writing Lean code, constructing proofs, fixing broken proofs. Works with the Lean compiler's feedback loop. Also integrates with Aeneas toolkit for verifying Rust code. Accepts text + images.
- **Why interesting:** As AI-generated code hits 42% of all commits (heading to 65% by 2027), the bottleneck shifts from *writing* code to *verifying* it. Leanstral is the first serious attempt at an AI agent that proves code correct, not just generates it. New FLTEval benchmark suite included. This is where "vibe coding" meets mathematical rigor.

### 2. MiroFish — Multi-Agent Swarm Prediction Engine
- **Link:** https://github.com/mirofish (hit #1 GitHub Global Trending in March 2026)
- **Stars:** Rapidly growing (thousands)
- **What:** Open-source multi-agent prediction engine that creates **simulated digital worlds** populated by up to 500K+ AI agents to predict market movements, public opinion, and social dynamics.
- **How:** Multiple interacting AI bots in a simulated environment, integrating real-world data feeds. Users run simulations with thousands of agents that develop behaviors and interact. Brian Roemmele reportedly ran 500K agents in a single simulation.
- **Why interesting:** Built by Guo Hangjiang, a senior undergrad in China. Attracted $4.3M investment from Shanda Group founder Chen Tianqiao. Novel approach — instead of one model predicting, you simulate an entire society of agents. Described as "scarily accurate" for market and public opinion prediction.

### 3. Apideck CLI — MCP Alternative That Uses 80 Tokens Instead of 55,000+
- **Link:** https://www.apideck.com/blog/mcp-server-eating-context-window-cli-alternative
- **Stars:** HN front page (#28, 131 points, 111 comments)
- **What:** CLI-based AI-agent interface that replaces MCP servers for API integration, using ~80 tokens of agent prompt vs MCP's 55,000+ token overhead.
- **How:** Progressive disclosure via `--help` flags. The agent gets a tiny prompt telling it about the CLI, then discovers capabilities on-demand through help commands rather than loading entire schemas upfront. Structural safety baked into the binary.
- **Why interesting:** Directly addresses MCP's biggest pain point — context window waste. Two orders of magnitude improvement. Part of a growing "CLI > MCP" movement. Relevant to OpenClaw's own architecture which uses CLI tools heavily.

### 4. The Agency — 61 Specialized AI Agents in One Repo
- **Link:** https://github.com/msitarzewski/agency-agents
- **Stars:** ~19.2K (10K in first 7 days)
- **What:** Collection of 61 AI specialist agents organized into 9 divisions, covering engineering, design, growth marketing, SEO, content, and more. Essentially a full AI creative/growth agency.
- **How:** Each agent has specialized system prompts, tool configurations, and workflows. Includes an installer that detects your coding tools (Claude Code, Copilot, Cursor, OpenClaw, etc.) and installs relevant agent configs. Not a framework — more of a curated prompt/config library with structure.
- **Why interesting:** Massive viral growth. Practical approach — instead of building one generalist, provide 61 specialists. The installer supporting multiple AI coding tools (including OpenClaw!) shows the multi-tool ecosystem is maturing.

### 5. Voygr (YC W26) — Maps API Built for AI Agents
- **Link:** https://voygr.com (Launch HN, 75 points)
- **What:** Mapping infrastructure designed specifically for AI agents — richer, more reliable place data than Google Maps for autonomous systems.
- **How:** API provides structured place data optimized for agent consumption rather than human map rendering. Addresses the problem of AI assistants recommending closed restaurants or outdated locations.
- **Why interesting:** Featured in Forbes' "21 Most Promising YC W26 Startups." Shows the infrastructure layer for agents is emerging — agents need different data formats than humans do. Maps are one of the first "agent-native" API redesigns.

## 🔑 Key Themes Today

1. **Verification > Generation:** Leanstral signals the shift from "can AI write code?" to "can AI prove code is correct?" Formal verification is the next frontier.
2. **MCP Backlash:** Apideck CLI and the "CLI > MCP" movement gaining steam. Context window waste is MCP's Achilles heel.
3. **Agent Simulation at Scale:** MiroFish shows multi-agent simulation (not just orchestration) is a viable prediction tool.
4. **Enterprise AI Governance:** Traefik, LangChain+NVIDIA — the plumbing for enterprise agent deployment is being built out rapidly.
5. **Agent-Native Infrastructure:** Voygr (maps), Apideck (APIs) — we're seeing the first wave of infrastructure redesigned for agent consumption rather than human consumption.
