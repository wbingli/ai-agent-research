# Tech Digest — 2026-05-09

**1. OpenAI’s WebRTC Problem**
A technical critique argues that realtime voice AI is exposing WebRTC’s complexity—NAT traversal, SDP, browser assumptions, and brittle media pipelines—and makes the case for Media over QUIC as a cleaner architecture. This matters for leaders building low-latency voice/video agents because transport choices can dominate reliability and operational complexity. [Read](https://moq.dev/blog/webrtc-is-the-problem/)

**2. OpenAI launches new realtime voice and translation AI models**
OpenAI introduced new realtime audio models for live voice agents, streaming transcription, and translation, pointing toward richer voice-first interfaces beyond chat boxes. The engineering takeaway is that latency, turn-taking, and continuous audio APIs are becoming core product infrastructure, not demos. [Read](https://www.testingcatalog.com/openai-launches-new-realtime-voice-and-translation-ai-models/)

**3. LLMs Corrupt Your Documents When You Delegate**
A new arXiv paper studies how LLM agents can unintentionally degrade documents when asked to edit or delegate work, a practical failure mode for agentic workflows. It is a reminder that autonomy needs diffing, provenance, rollback, and human review loops before teams trust agents with important artifacts. [Read](https://arxiv.org/abs/2604.15597)

**4. Bun’s experimental Rust rewrite hits 99.8% test compatibility**
Bun’s Rust rewrite reportedly passes 99.8% of its existing Linux x64 glibc test suite, suggesting a major runtime implementation shift is nearing practical parity. For platform teams, this is a useful signal about the JavaScript tooling ecosystem’s direction: performance and maintainability pressure are pushing core infrastructure toward Rust. [Read](https://twitter.com/jarredsumner/status/2053047748191232310)

**5. Stop MITM on the first SSH connection, on any VPS or cloud provider**
This writeup explains a practical way to verify SSH host keys on first connection using provider console access, closing the classic “trust on first use” gap. It is a small but valuable hardening pattern for infra teams provisioning fleets across commodity VPS and cloud providers.
