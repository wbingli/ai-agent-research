# Tech Digest — 2026-07-10

**1. Rewriting Bun in Rust**
Bun explains why it is moving large parts of its JavaScript runtime/toolchain from Zig to Rust, a notable case study in language choice, ecosystem leverage, and maintainability under production pressure. For engineering leaders, it is a reminder that “fast systems language” decisions are also about hiring, libraries, tooling, and long-term operational velocity. [Read](https://bun.com/blog/bun-in-rust)

**2. OpenAI launches GPT-5.6 Sol, Terra, and Luna on apps and API**
OpenAI’s new GPT-5.6 family reportedly spans higher-intelligence and faster variants across ChatGPT, Codex, and the API, continuing the move toward model portfolios optimized by task and latency/cost profile. Teams building AI products should expect more routing, eval, and cost-control work rather than a single default model choice. [Read](https://www.testingcatalog.com/openai-launches-gpt-5-6-sol-terra-and-luna-on-apps-and-api/)

**3. Apple Exploring Ways to Run Much Larger AI Models Directly on iPhones**
Apple is reportedly exploring PrismML technology to run much larger compressed models directly on iPhones, including a 27B-parameter Qwen variant. If practical, this pushes more AI capability to the edge, with big implications for privacy, latency, offline UX, and cloud inference spend. [Read](https://www.macrumors.com/2026/07/09/apple-prismml-larger-on-device-ai-models/)

**4. How GitHub gave every repository a durable owner**
GitHub describes a security architecture change that ensures every repository always has a durable owner, reducing orphaned-resource risk and tightening authorization boundaries. This is the kind of unglamorous identity and ownership plumbing that prevents large platforms from accumulating dangerous edge cases over time. [Read](https://github.blog/security/application-security/how-github-gave-every-repository-a-durable-owner/)

**5. Postgres rewritten in Rust, now passing 100% of the Postgres regression tests**
A project claims a Rust rewrite of Postgres that passes the full upstream regression suite, making it an interesting experiment in database compatibility and automated translation/reimplementation. Even if far from production, it is a useful signal for where systems-language migration, test-suite fidelity, and database internals research are heading. [Read](https://github.com/malisper/pgrust)
