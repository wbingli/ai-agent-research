# Tech Digest — 2026-06-10

**1. How building an HTML-first site doubled our users overnight**
A utility site rebuilt around server-rendered HTML, less JavaScript, and faster core flows reportedly doubled usage overnight. It is a useful reminder that product growth can come from reducing client complexity, improving accessibility, and making the default path fast. [Read](https://mohkohn.co.uk/writing/html-first/)

**2. Cybersecurity researchers aren't happy about the guardrails on Anthropic's Fable**
Security researchers say Anthropic's new Fable model refuses too much legitimate offensive-security and vulnerability research work. For engineering leaders, the tension is operational: AI safety policies can become a platform reliability and productivity constraint for security teams. [Read](https://techcrunch.com/2026/06/10/cybersecurity-researchers-arent-happy-about-the-guardrails-on-anthropics-fable/)

**3. Claude Desktop spawns 1.8 GB Hyper-V VM on every launch, even for chat-only use**
A GitHub issue reports Claude Desktop starting a large Hyper-V VM even when users only want chat, raising concerns about resource use, transparency, and enterprise endpoint behavior. AI desktop tools are becoming infrastructure, so their sandboxing and background runtime costs need the same scrutiny as any developer platform dependency. [Read](https://github.com/anthropics/claude-code/issues/29045)

**4. PgDog is funded and coming to a database near you**
PgDog is a PostgreSQL connection pooler, load balancer, and sharding proxy aimed at horizontal scale without forcing application rewrites. It matters because many teams want more headroom from Postgres while postponing the operational and product complexity of a full database migration. [Read](https://pgdog.dev/blog/our-funding-announcement)

**5. Apache Burr: Build reliable AI agents and applications**
Apache Burr is a Python framework for building AI applications as explicit state machines, with decisions and transitions modeled rather than hidden in ad hoc chains. That framing is valuable for production agent systems because reliability, debugging, and governance improve when behavior is inspectable. [Read](https://burr.apache.org/)
