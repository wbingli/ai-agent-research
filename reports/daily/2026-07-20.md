# Tech Digest — 2026-07-20

**1. How Anthropic runs large-scale code migrations with Claude Code**
Anthropic describes a six-step agent workflow behind migrations including Bun’s million-line Zig-to-Rust port and a 165,000-line Python-to-TypeScript rewrite. The leadership takeaway: phase gates, adversarial review, behavioral parity checks, and strong test suites—not raw code generation—make high-scale AI migrations viable. [Read](https://claude.com/blog/ai-code-migration)

**2. The week of coding-agent sandbox escapes**
Researchers reproduced seven sandbox escapes and boundary bypasses across Cursor, Codex, Gemini CLI, and Antigravity, often by having an agent write files that trusted host components later executed or loaded. Security reviews must therefore cover the agent’s full writable trust graph, not just its process boundary. [Read](https://www.pillar.security/blog/the-week-of-sandbox-escapes)

**3. Human mathematicians are being outcounterexampled**
AI systems are producing counterexamples to open mathematical conjectures and, increasingly, formalizing the results in Lean—including work that generated 1.2 million lines of proof code in three weeks. This points toward a powerful model for high-assurance engineering: machine discovery paired with machine-checkable verification. [Read](https://xenaproject.wordpress.com/2026/07/20/human-mathematicians-are-being-outcounterexampled/)

**4. Agent swarms and the new model economics**
Cursor’s redesigned swarm built a Rust implementation of SQLite more accurately and cheaply than its earlier system, with better coordination outperforming simply adding model capacity. For engineering leaders, orchestration architecture and cost controls may matter as much as choosing the strongest model. [Read](https://cursor.com/blog/agent-swarm-model-economics)

**5. Opaque, interoperable passkey records**
A proposed standard packages WebAuthn credential state into portable opaque records, giving applications a password-hash-like abstraction while preserving passkeys’ phishing resistance. Standardizing this storage layer could reduce implementation complexity and vendor lock-in for passwordless authentication. [Read](https://words.filippo.io/passkey-record/)
