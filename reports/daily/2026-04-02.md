# AI Agent Research Daily Digest — April 2, 2026

## 🔥 Top Discoveries

### 1. Oh My Codex (OMX) — Workflow Layer for OpenAI Codex CLI
- **Link:** https://github.com/Yeachan-Heo/oh-my-codex
- **Stars:** 10,439 ⭐ (+2,852 today)
- **What:** A workflow orchestration layer that wraps OpenAI's Codex CLI. Adds structured workflows (`$deep-interview` for clarification, `$ralplan` for plan approval, `$ralph` for persistent completion, `$team` for parallel agent execution), project state management (`.omx/` directory), and reusable specialist roles.
- **How:** npm package that installs alongside Codex CLI. Uses tmux for durable multi-agent team mode. Keeps plans, logs, memory, and mode tracking in project-local state.
- **Why interesting:** This is the "oh-my-zsh for coding agents" pattern — taking a powerful but raw CLI tool and adding opinionated workflows around it. The structured clarify→plan→approve→execute pipeline is a practical answer to the "just point an agent at code" anti-pattern. The parallel `$team` mode using tmux for multi-agent coordination is clever. OpenClaw integration guide included.

### 2. PrismML 1-Bit Bonsai — First Commercially Viable 1-Bit LLMs
- **Link:** https://prismml.com/ | [Show HN](https://news.ycombinator.com/item?id=47593422) (409 pts)
- **What:** Family of 1-bit weight LLMs (8B, 4B, 1.7B params). The 8B model fits in 1.15GB RAM — 14× smaller than full-precision, 8× faster, 5× more energy efficient, while matching leading 8B models on benchmarks.
- **How:** Novel quantization-aware training that produces true 1-bit weights (not post-training quantization). The 1.7B model hits 130 tok/s on iPhone 17 Pro Max with only 0.24GB footprint.
- **Why interesting:** This could be transformative for on-device AI agents. Running a competitive 8B model in ~1GB RAM means real agent capabilities on phones, Raspberry Pis, and edge devices. The "10× intelligence density" claim, if it holds, changes the economics of local inference. Perfect for robotics and real-time agents where latency matters more than peak capability.

### 3. GLM-OCR — State-of-the-Art Document OCR in 0.9B Parameters
- **Link:** https://github.com/zai-org/GLM-OCR
- **Stars:** 5,155 ⭐ (+238 today)
- **What:** Multimodal OCR model built on GLM-V encoder-decoder architecture. Scores #1 on OmniDocBench V1.5 (94.62). Handles formulas, tables, code, seals, and complex layouts. Only 0.9B parameters.
- **How:** Uses Multi-Token Prediction loss + stable full-task RL training. CogViT visual encoder with token downsampling + GLM-0.5B language decoder. Two-stage pipeline: layout analysis (PP-DocLayout-V3) → parallel region recognition. Deployable via vLLM, SGLang, or Ollama.
- **Why interesting:** Tiny model, SOTA results. The agent-friendly SDK mode (pip install + CLI) makes it trivially integrable into agentic workflows — imagine coding agents that can read screenshots, PDFs, or whiteboard photos. Could replace expensive vision API calls for document understanding tasks.

### 4. Microsoft VibeVoice — Open-Source Frontier Voice AI
- **Link:** https://github.com/microsoft/VibeVoice
- **Stars:** 35,019 ⭐ (+1,085 today)
- **What:** Family of open-source voice AI models covering TTS (text-to-speech) and ASR (speech recognition). The ASR model handles 60-minute long-form audio in a single pass with speaker diarization, timestamps, and custom context. The real-time TTS (0.5B) supports streaming input and 9+ languages.
- **How:** Now integrated into HuggingFace Transformers (v5.3.0). ASR produces structured output: Who (speaker) + When (timestamp) + What (content). Fine-tuning code available. Community already building on top (Vibing — voice-powered input method).
- **Why interesting:** Open-source voice AI reaching frontier quality. The 60-min single-pass ASR with speaker identification is huge for meeting transcription, podcast processing, and voice agent backends. Already spawning ecosystem tools (Vibing). Accepted as ICLR 2026 Oral.

### 5. TinyLoRA — Learning to Reason in 13 Parameters
- **Link:** https://arxiv.org/abs/2602.04118 | [HN](https://news.ycombinator.com/item?id=47541733) (231 pts)
- **What:** Research paper demonstrating that reasoning capabilities can be encoded in extremely small LoRA adapters — as few as 13 trainable parameters.
- **Why interesting:** Challenges assumptions about the scale needed for reasoning. If reasoning can be captured in tiny adapters, it suggests reasoning is more about activating latent capabilities than adding new ones. Implications for efficient fine-tuning and understanding what LLMs actually learn.
