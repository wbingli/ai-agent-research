# Tech Digest — 2026-06-25

**1. How we built the world’s fastest API for GLM-5.2**
Baseten breaks down an AI inference stack hitting 280+ tokens/sec on NVIDIA Blackwell using KV-aware routing, prefill/decode disaggregation, multi-token prediction, and NVFP4. Useful for leaders thinking about where LLM serving performance now comes from: not one trick, but tight coordination across model, scheduler, routing, and hardware. [Read](https://www.baseten.co/blog/how-we-built-the-worlds-fastest-api-for-glm-52/#higher-tps-with-multi-token-prediction)

**2. Announcing Silk: a silky smooth fiber runtime for ClickHouse**
ClickHouse introduced Silk, a C++ fiber runtime built around NUMA-aware work stealing, io_uring I/O, and zero steady-state heap allocation. It is a reminder that database tail latency still depends on low-level runtime design, especially as analytical systems handle more concurrent mixed workloads. [Read](https://clickhouse.com/blog/silk)

**3. Scaling Rails: 41M Req/Hour, 8 DBs, disable_joins: true**
Aura Frames describes how it scaled a Rails app for holiday peak load by splitting across eight primary databases and leaning on Rails multiple-database support, including replacing cross-DB joins with application-level reads. The useful lesson is pragmatic: mature monolith stacks can scale far when teams understand their data boundaries and operational peak patterns. [Read](https://andyatkinson.com/how-aura-frames-scales-for-peak-load-ruby-on-rails)

**4. OpenAI and Broadcom reveal Jalapeño, first AI chip in partnership**
OpenAI and Broadcom disclosed their first custom AI-chip project, another sign that frontier AI companies are moving deeper into silicon and supply-chain control. For engineering leaders, this reinforces that AI platform strategy is increasingly about capacity planning, vendor leverage, and hardware-software co-design—not just model choice. [Read](https://www.cnbc.com/2026/06/24/openai-and-broadcom-reveal-jalapeno-first-ai-chip-in-partnership.html)

**5. I wrote a 70x faster SQL parser while barely looking at the code**
PostHog recounts using coding agents to rewrite a SQL parser for a major speedup, with humans steering benchmarks and review rather than every implementation detail. The takeaway is practical: agentic coding is strongest when wrapped in tight feedback loops, tests, and measurable performance targets. [Read](https://posthog.com/blog/sql-parser)
