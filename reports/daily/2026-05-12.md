# Tech Digest — 2026-05-12

**1. Introducing Claude Platform on AWS**
AWS is now offering Anthropic’s native Claude Platform directly through AWS accounts, including Anthropic APIs, console experience, managed agents, and AWS billing/identity integration. For engineering leaders, this lowers procurement and governance friction for enterprise agent deployments while keeping teams inside existing cloud controls. [Read](https://aws.amazon.com/blogs/machine-learning/introducing-claude-platform-on-aws-anthropics-native-platform-through-your-aws-account/)

**2. Quack: The DuckDB Client-Server Protocol**
DuckDB introduced Quack, a remote protocol that lets DuckDB instances talk over HTTP and support client-server use cases with multiple concurrent writers. This is notable because DuckDB keeps moving from embedded analytics into lightweight operational/data-platform territory without adopting heavyweight database machinery. [Read](https://duckdb.org/2026/05/12/quack-remote-protocol)

**3. When “idle” isn’t idle: how a Linux kernel optimization became a QUIC bug**
Cloudflare traced a severe QUIC throughput collapse to congestion-control accounting that confused application idleness with RTT wait time, pinning CUBIC’s congestion window at its floor. It is a useful reminder that performance bugs in distributed systems often emerge at the boundary between kernel behavior, transport algorithms, and production traffic patterns. [Read](https://blog.cloudflare.com/quic-death-spiral-fix/)

**4. TanStack npm packages compromised in ongoing mini Shai-Hulud supply-chain attack**
Socket reports that 84 TanStack npm package artifacts were modified with suspected CI credential-stealing malware, affecting packages with very high weekly download counts. This is a sharp signal that dependency risk is increasingly about build and release pipelines, not just vulnerable code in the repository. [Read](https://socket.dev/blog/tanstack-npm-packages-compromised-mini-shai-hulud-supply-chain-attack)

**5. Mythos finds a curl vulnerability**
Daniel Stenberg’s writeup gives a grounded account of Anthropic’s Mythos finding a real curl vulnerability, but also shows why “AI found a bug” needs careful validation and context. For leaders evaluating AI security tooling, the takeaway is to measure actual triage value and false-positive cost rather than buying the headline. [Read](https://daniel.haxx.se/blog/2026/05/11/mythos-finds-a-curl-vulnerability/)
