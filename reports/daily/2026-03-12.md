# AI Agent Research Daily Digest — 2026-03-12

## 🔥 Top Discoveries

### 1. Rudel — Claude Code Session Analytics
- **Link:** https://github.com/obsessiondb/rudel
- **What:** Analytics dashboard for Claude Code sessions — token usage, session duration, activity patterns, model usage, sub-agent tracking
- **How:** CLI hooks into Claude Code's session end event, uploads transcripts to ClickHouse, processes into analytics. Hosted version at rudel.ai
- **Why interesting:** As coding agents become daily drivers, understanding *how* you use them (cost, patterns, which models for what) becomes critical. First serious observability tool specifically for Claude Code. Currently on HN front page with strong engagement (Show HN)
- **Caveat:** Uploads full session transcripts including source code — privacy-sensitive

### 2. Junie CLI — JetBrains' LLM-Agnostic Coding Agent (Beta)
- **Link:** https://blog.jetbrains.com/junie/2026/03/junie-cli-the-llm-agnostic-coding-agent-is-now-in-beta/
- **What:** JetBrains' standalone coding agent, now works from terminal (not just IDE). LLM-agnostic — supports OpenAI, Anthropic, Google, Grok
- **How:** BYOK pricing (bring your own API key), one-click migration from Claude Code/Codex, native MCP support with auto-detection of useful MCP servers, real-time prompt refinement during execution, next-task prediction
- **Why interesting:** JetBrains brings deep code intelligence (not just "AI in a terminal"). The MCP auto-recommendation feature is novel — agent suggests relevant MCP servers for your task. Free Gemini 3 Flash access for first week. Direct competitor to Claude Code and Codex CLI

### 3. Axe — Unix-Philosophy AI Agent Runner
- **Link:** https://github.com/jrswab/axe
- **Stars:** New (Show HN today)
- **What:** 12MB Go binary that treats LLM agents like Unix programs — each does one thing well. Define agents in TOML, pipe data in/out, chain together
- **How:** TOML-based agent configs, multi-provider (Anthropic/OpenAI/Ollama), sub-agent delegation via tool use, persistent markdown memory, stdin piping (`git diff | axe run reviewer`), built-in file/shell tools sandboxed per agent. Two direct Go dependencies total
- **Why interesting:** Radically minimal philosophy vs. bloated frameworks. Composable with existing Unix tools (cron, git hooks, CI). The "anti-framework" approach — no daemon, no GUI, just a binary and configs. Feels like the `jq` of AI agents

### 4. Agency-Agents — 112 Specialized AI Agent Personas
- **Link:** https://github.com/msitarzewski/agency-agents
- **Stars:** 10K+ in 7 days
- **What:** Collection of 112 specialized AI agent persona definitions spanning engineering, design, marketing, product, support, spatial computing, and project management
- **How:** Each agent defined with personality, processes, and deliverables. Works as prompt library for coding agents (Claude Code, Cursor, etc.). Includes examples for multi-agent coordination
- **Why interesting:** Not a framework — it's a curated set of agent *identities*. The idea that specialized personas with domain expertise and personality outperform generic assistants. 10K stars in a week suggests this resonates. Useful as inspiration for OpenClaw skill definitions

### 5. GitHub Copilot SDK — Agentic Execution as a Library
- **Link:** https://github.com/github/copilot-sdk/
- **Blog:** https://github.blog/ai-and-ml/github-copilot/the-era-of-ai-as-text-is-over-execution-is-the-new-interface/
- **What:** Embed the same planning and execution engine that powers GitHub Copilot CLI into your own applications. Programmable agentic workflows
- **How:** SDK provides the orchestration layer (planning, tool invocation, file modification, error recovery) so you don't build your own. Three patterns: autonomous pipelines, human-in-the-loop, and hybrid
- **Why interesting:** GitHub's thesis: "AI as text is over, execution is the new interface." This democratizes agentic capabilities — any app can now have Copilot-grade agent execution. Potential game-changer for building internal tools with AI agents
