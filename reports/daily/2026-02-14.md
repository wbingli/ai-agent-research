### 3. CodeGraphContext â€” MCP Server Using Graph Databases for Code Context
- **Link:** https://codegraphcontext.github.io/ | https://www.reddit.com/r/mcp/comments/1qyncmd/
- **What:** An MCP server that indexes your codebase into a graph database (not vector embeddings) to provide structural code context to AI assistants. Uses AST parsing to build a graph of functions, classes, imports, call chains. Offers CLI tools (`cgc watch` for live tracking) and exposes context via MCP protocol.
- **Why interesting:** This is a direct challenge to the dominant "chunk-and-embed" RAG approach for code. The r/mcp community response confirms this resonates: "chunk-RAG turns into token spam fast." Persistent graph DB means incremental updates, not re-indexing.
- **Relevance:** Novel approach to the code context problem that every coding agent faces. Could be integrated with Claude Code, Cline, etc. via MCP.
