# Tech Digest — 2026-07-01

**1. Hundreds of AUR packages attacked by infostealer**
Attackers compromised hundreds of Arch User Repository packages with credential-stealing malware, a sharp reminder that community package ecosystems are production supply-chain risk. Engineering leaders should treat developer workstations, CI builders, and “optional” package feeds as part of the security perimeter. [Read](https://lists.archlinux.org/archives/list/aur-general@lists.archlinux.org/thread/FGXPCB3ZVCJIV7FX323SBAX2JHYB7ZS4/)

**2. Cloudflare’s Monetization Gateway brings x402 payments to protected resources**
Cloudflare introduced a way to charge for access to APIs, files, or other resources behind Cloudflare using the x402 payment protocol. If it gains traction, it could make metered machine-to-machine commerce simpler for API products, agents, datasets, and premium infrastructure services. [Read](https://blog.cloudflare.com/monetization-gateway/)

**3. Google open-sources zero-knowledge proof libraries for age assurance**
Google released ZKP libraries aimed at proving age eligibility without revealing the user’s exact birthdate or identity details. This is relevant beyond age checks: privacy-preserving verification is becoming a core architecture pattern for regulated consumer platforms. [Read](https://blog.google/innovation-and-ai/technology/safety-security/opening-up-zero-knowledge-proof-technology-to-promote-privacy-in-age-assurance/)

**4. IPFS content publishing gets 10x faster with Optimistic Provide**
ProbeLab describes a statistical “Optimistic Provide” approach that cuts IPFS publishing latency by more than an order of magnitude. The broader lesson is useful for distributed systems teams: carefully relaxing certainty at the protocol edge can unlock major performance wins without abandoning correctness goals. [Read](https://probelab.io/blog/optimistic-provide/)

**5. ZCode ships as an official coding harness for GLM-5.2**
Z.ai launched ZCode, a coding-agent environment positioned around GLM-5.2 for planning, coding, reviewing, and deploying inside existing developer workflows. It is another signal that frontier and open-weight model vendors are competing through full engineering harnesses, not just raw model APIs. [Read](https://zcode.z.ai/en)
