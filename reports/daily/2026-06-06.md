# Tech Digest — 2026-06-06

**1. Meta confirms thousands of Instagram accounts were hacked by abusing its AI chatbot**
Meta fixed a bug where attackers could trick Meta AI into resetting passwords for Instagram accounts without 2FA. It is a sharp reminder that AI assistants wired into account workflows become security-critical control planes, not just chat UX. [Read](https://this.weekinsecurity.com/meta-confirms-thousands-of-instagram-accounts-were-hacked-by-abusing-its-ai-chatbot/)

**2. Harness engineering: leveraging Codex in an agent-first world**
OpenAI frames “harness engineering” as the work of building repos, tests, sandboxes, evals, and review loops so coding agents can operate safely and usefully. For engineering leaders, the leverage is less about prompting and more about making the software delivery system agent-ready. [Read](https://openai.com/index/harness-engineering/)

**3. The Smart TV in Your Living Room Is a Node in the AI Scraping Economy**
Include Security shows how Bright Data’s SDK can turn consumer TVs into residential proxy exit nodes used for large-scale web scraping. The story connects privacy, supply-chain risk, and AI data acquisition: seemingly harmless embedded apps can become infrastructure for someone else’s crawler fleet. [Read](https://blog.includesecurity.com/2026/06/the-smart-tv-in-your-livingroom-is-a-node-in-the-aiscraping-economy/)

**4. Did Claude Increase Bugs in rsync?**
A detailed analysis looks at whether AI-assisted changes in rsync correlated with new bugs, using commit history and bug reports rather than vibes. It is useful because teams adopting coding agents need measurement disciplines that separate productivity gains from hidden maintenance and security costs. [Read](https://alexispurslane.github.io/rsync-analysis/)

**5. Running Python code in a sandbox with MicroPython and WASM**
Simon Willison demonstrates a lightweight way to run untrusted Python by compiling MicroPython to WebAssembly and constraining execution. This matters for agent platforms, notebooks, and internal tools that need programmable extensions without handing arbitrary code full host access. [Read](https://simonwillison.net/2026/Jun/6/micropython-in-a-sandbox/)
