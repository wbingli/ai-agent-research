# Tech Digest — 2026-07-16

**1. Kimi K3: Open Frontier Intelligence**
Moonshot released Kimi K3, an open 3T-class model claiming frontier-level coding, reasoning, multimodality, and 1M-token context. For engineering leaders, the important signal is that open-weight systems are continuing to pressure closed-model capability, deployment flexibility, and cost assumptions. [Read](https://www.kimi.com/blog/kimi-k3)

**2. You only need the frontier model for one single edit**
Stencil describes “prewalk,” a workflow where a frontier model produces the trajectory once, then cheaper models execute most of the edits while retaining much of the pass rate. This is a practical pattern for agentic coding economics: spend premium inference on search/planning, not every token of implementation. [Read](https://stencil.so/blog/prewalk)

**3. NotebookLM is now Gemini Notebook**
Google is rebranding NotebookLM as Gemini Notebook and tying it more deeply into Google’s Gemini stack, including secure cloud-computer capabilities. The move shows document-grounded AI assistants evolving from research tools into enterprise productivity surfaces with stronger workflow integration. [Read](https://blog.google/innovation-and-ai/products/gemini-notebook/notebooklm-gemini-notebook/)

**4. How Our Rust-to-Zig Rewrite Is Going**
Richard Feldman shares lessons from rewriting a large Rust codebase in Zig, with concrete tradeoffs around compile times, language complexity, safety, and day-to-day developer experience. It is useful reading for leaders weighing whether systems-language choices are strategic architecture decisions or local optimization traps. [Read](https://rtfeldman.com/rust-to-zig)

**5. We’re building Postgres in Rust. Using the LLVM of databases**
Turso says it is extending its Rust database-core strategy from SQLite toward Postgres, aiming for a reusable engine that can support multiple SQL frontends. If it works, it points to a future where database compatibility layers and storage engines are composed more like compiler toolchains. [Read](https://turso.tech/blog/a-new-modern-version-of-postgres-in-rust)
