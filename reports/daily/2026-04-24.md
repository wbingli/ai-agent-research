### 1. Introducing GPT-5.5 ("Spud")
- **Source:** Hacker News (front page) / OpenAI
- **Topic:** AI / Frontier Models / Agentic Orchestration
- **Link:** [OpenAI — Introducing GPT-5.5](https://openai.com/index/introducing-gpt-5-5/)
- **Summary:** OpenAI shipped GPT-5.5 on April 23 — their first fully retrained base model since GPT-4.5. Natively omnimodal (text/image/audio/video in one system), more token-efficient, and explicitly designed for agentic multi-tool orchestration. Priced at $5/$30 per 1M input/output tokens (double GPT-5.4).
- **Why it's relevant:** This is the model that will show up under every agent framework in the next quarter. The "designed for multi-tool orchestration" framing matters for any agentic ad-ops/targeting work — the routing rule emerging in the wild is *agentic + computer use → GPT-5.5, coding → Opus 4.7, bulk/cheap → Haiku 4.5 or GPT-5.4 mini*.

- **Source:** Hacker News / GitHub Changelog
- **Topic:** AI / Coding Agents
- **Link:** [GitHub — Claude Opus 4.7 is generally available](https://github.blog/changelog/2026-04-16-claude-opus-4-7-is-generally-available/)

### 4. Vercel April 2026 security incident (Context.ai supply chain)
- **Source:** Hacker News / Vercel KB
- **Topic:** AI Supply-Chain Security
- **Link:** [Vercel Knowledge Base — April 2026 security incident](https://vercel.com/kb/bulletin/vercel-april-2026-security-incident)
- **Summary:** Confirmed attack chain: Lumma Stealer infection at Context.ai (a third-party AI tool) → stolen Google Workspace OAuth tokens → pivot into a Vercel employee's account → access to environment variables for a limited subset of customer projects. - **Why it's relevant:** This is the first high-profile "AI vendor as the supply-chain attack vector" incident we've seen with this level of detail. Worth running the incident-response playbook mentally for any third-party AI/agent vendor we integrate — the blast radius is OAuth tokens and env vars, not just model output.

### 5. 175,000 publicly exposed Ollama hosts — "Operation Bizarre Bazaar"
- **Source:** Hacker News / SentinelLABS + Censys
- **Topic:** AI Infrastructure Security
- **Link:** [The Hacker News — 175,000 publicly exposed Ollama servers](https://thehackernews.com/2026/01/researchers-find-175000-publicly.html)
- **Summary:** A 293-day joint scan found ~175K Ollama instances across 130 countries (most in China) exposed without auth. ~half have tool-calling enabled, turning them into free remote-code-execution endpoints. Active LLMjacking monetization campaign dubbed *Operation Bizarre Bazaar*.
- **Why it's relevant:** The "unmanaged AI compute layer" is now a real attack surface class, not a thought experiment. Relevant framing for any internal self-hosted inference — and a useful war-story to bring up whenever someone proposes binding an LLM server to `0.0.0.0` "just for testing."

### 6. Meta — Investing in Infrastructure: Renewed Commitment to jemalloc
- **Source:** Lobste.rs ("Who even uses jemalloc in 2026 anyway?") / Engineering at Meta
- **Topic:** Systems / Memory Allocators at Scale
- **Link:** [Engineering at Meta — Renewed commitment to jemalloc](https://engineering.fb.com/2026/03/02/data-infrastructure/investing-in-infrastructure-metas-renewed-commitment-to-jemalloc/)
- **Summary:** Meta is re-investing in jemalloc after a period of uncertainty — modernizing the codebase, improving the hugepage allocator (HPA) for transparent hugepage utilization, and pushing for strong out-of-the-box AArch64 performance. Adjacent benchmark piece shows mimalloc ~15% better P99 than jemalloc for tiny-alloc workloads, but the gap narrows to <3% for long-running stable workloads.
- **Why it's relevant:** Ad-serving is *exactly* the "long-running stable memory pattern" case where jemalloc shines. If we're still on glibc malloc for any hot path, the THP/HPA story here is worth a look — typical wins are single-digit CPU percent at ad-tech-scale QPS, which is real money.

### 7. Simon Willison — Changes in the system prompt between Claude Opus 4.6 and 4.7
- **Source:** simonwillison.net (Apr 18)
- **Topic:** LLM Tooling / Prompt Engineering
